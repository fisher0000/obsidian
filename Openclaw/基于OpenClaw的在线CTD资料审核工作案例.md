# 基于OpenClaw的在线CTD资料审核工作案例

## 1. 目的

### 1.1 背景与需求
在药品注册申报过程中，CTD（Common Technical Document）资料的审核是保证申报质量的关鍵环节。传统人工审核存在效率低、主观性强、难以标准化等问题。本案例旨在通过**OpenClaw AI平台**，建立一套智能化的在线CTD资料审核工作流程，实现高效、合规、可追溯的注册资料质量管控。

### 1.2 具体目标
- **提升审核效率**：利用OpenClaw的飞书文档集成能力，实现秒级文档访问和分析
- **确保法规合规**：依托OpenClaw内置的ICH指导原则知识库，提供标准化审核
- **降低人工差错**：通过AI系统性检查，减少人为疏漏
- **建立知识沉淀**：利用OpenClaw的Memory功能，持续积累审核经验和最佳实践
- **实现团队协作**：借助OpenClaw的多渠道能力，支持跨部门协作和结果分享

## 2. 基础条件

### 2.1 OpenClaw平台配置要求

| 配置项 | 要求 | 说明 |
|--------|------|------|
| **OpenClaw版本** | ≥ 0.24.x | 支持飞书文档、飞书wiki、Bitable等集成 |
| **技能模块** | feishu-doc, feishu-wiki | 用于读取和分析飞书文档 |
| **Memory系统** | 已启用 | 支持审核经验的长期存储和复用 |
| **文件系统权限** | 可写入本地或网络路径 | 用于存储审核报告 |

### 2.2 文档环境准备

#### 飞书文档准备
- 将Word/PDF格式的CTD资料转换为**飞书在线文档**（docx格式）
- 设置权限为"组织内可查看"或"任何人可查看"
- 确保文档token可从链接中提取（format: `https://xxx.feishu.cn/docx/TOKEN`）

#### 文档格式规范
```
✓ 推荐使用：飞书文档（docx）
✗ 不推荐使用：飞书文件（file/*）
说明：OpenClaw的feishu-doc技能仅支持docx格式直接读取
```

### 2.3 技能和工具配置

**必需技能：**
- `feishu-doc` - 飞书文档读取（支持CTD资料直接读取）
- `feishu-wiki` - 飞书知识库文档访问
- `write`/`edit` - 文件操作（生成和保存审核报告）

**可选技能：**
- `memory_search` - 记忆检索（查询历史审核经验）
- `web_search` - 网络搜索（查询最新的法规要求）
- `notion` - 多维表格更新（用于任务跟踪）

## 4. 详细操作步骤

### 4.1 步骤一：准备审核请求（用户操作）

#### 4.1.1 转换文档格式（如必要）
如果CTD资料是Word或PDF格式，需要先转换为飞书文档：

1. 打开飞书云文档
2. 上传原始文件
3. 右键点击 → "转换为在线文档"
4. 获取分享链接（确保是 docx 格式链接）

#### 4.1.2 发起审核请求
通过飞书私信向OpenClaw发送审核请求：

```
CTD资料的[模块名称]飞书链接：[飞书docx链接]
重点关注：[具体的审核要求]
请以[格式]输出审核结果到[路径]
```

**示例：**
```
CTD资料的物料控制部分飞书链接：
https://xxx.feishu.cn/docx/AbCdEfGhI123

重点关注：
- 检查起始原料选择是否符合ICH Q11
- 质量标准是否完整
- 方法学验证资料是否齐全

请以Markdown格式输出审核结果到D:\out文件夹
```

### 4.2 步骤二：OpenClaw自动接入文档

#### 4.2.1 文档链接解析
OpenClaw自动从链接中提取文档token：

```
链接格式：https://xxx.feishu.cn/docx/R6uRdZFMco0qwGxTOhPcWHHAnjX
文档token：R6uRdZFMco0qwGxTOhPcWHHAnjX
```

#### 4.2.2 调用feishu-doc技能读取内容
OpenClaw自动执行：
```json
{
  "action": "read",
  "doc_token": "R6uRdZFMco0qwGxTOhPcWHHAnjX"
}
```

**成功返回：**
- 文档title
- 完整content内容
- block_count和block_types统计

**失败处理：**
- 如果返回404错误 → 提示用户检查文档权限
- 如果返回file格式不支持 → 提示用户转换为docx格式

### 4.3 步骤三：AI智能分析审核

#### 4.3.1 审核内容分类
根据CTD模块类型，OpenClaw自动应用相应的审核标准：

| CTD模块 | 审核重点 | OpenClaw检查清单 |
|---------|---------|-----------------|
| 3.2.S.2.2 工艺控制 | 工艺流程图、关键参数 | 工艺完整性、参数合理性、命名一致性 |
| 3.2.S.2.3 物料控制 | 起始原料、试剂、溶剂 | ICH Q11合规性、质量标准、方法验证 |
| 3.2.S.4 质量控制 | 质量标准、分析方法 | 药典符合性、验证完整性 |

#### 4.3.2 检查问题分类
OpenClaw将识别的问题自动分类：

```
🔴 Critical（重大缺陷）- 影响注册成功的关键问题
- 干燥温度描述错误（如"6045～8050℃"）
- 批量范围描述矛盾
- 关键工艺参数表格缺失

🟠 Major（一般缺陷）- 需要修正的问题
- 工艺流程图缺少关键参数标注
- 化合物命名不一致
- 无菌工艺关键参数不完整

🟡 Minor（建议项）- 优化建议
- 增加异常情况处理程序
- 补充中间产品放行标准
```

#### 4.3.3 Memory系统应用
OpenClaw可查询历史审核经验：

```
Query: "头孢类抗生素CTD工艺控制常见问题"
Result: 
- 温度参数范围不一致是高发问题
- 化合物命名统一是重要的审核点
- 批量范围需要明确的计算依据
```

### 4.4 步骤四：生成审核报告

#### 4.4.1 报告结构设计

## 5. 注意事项

### 5.1 OpenClaw使用注意事项

#### 5.1.1 文档格式兼容性
```
✓ 支持的格式：
  - 飞书在线文档（docx/*）
  - 飞书Wiki页面
  
✗ 不支持的格式：
  - 飞书文件下载链接（file/*）
  - 外部云盘链接
  - 本地文件路径

解决方案：
如需审核非飞书docx格式的文件，请先在飞书中
"转换为在线文档"后再分享链接。
```

#### 5.1.2 链接格式处理
- **有效链接格式：** `https://xxx.feishu.cn/docx/TOKEN`
- **无效链接格式：** `https://xxx.feishu.cn/file/TOKEN`（文件下载链接）

区分方法：链接中包含`/docx/`为支持的在线文档格式。

#### 5.1.3 文件存储路径
- **Windows路径格式：** `D:\out\` 或 `D:/out/`
- **路径存在性：** OpenClaw会自动创建不存在的父目录
- **文件覆盖：** 如果文件已存在，会覆盖原有内容

### 5.2 审核质量注意事项

#### 5.2.1 AI审核的局限性
- **需要人工复核：** AI识别的问题应由专业人员最终确认
- **法规更新：** 确保OpenClaw的法规数据库是最新的
- **上下文理解：** 复杂的工艺问题可能需要结合专业知识判断

#### 5.2.2 数据完整性检查
审核文档前应确认：
- 所有引用的表格和图表是否完整
- "详见附件XX"之类的引用是否提供了附件
- 数据表格中的"XXX"占位符是否已补充实际数据

### 5.3 常见问题及解决方案

| 问题 | 原因 | OpenClaw解决方案 |
|------|------|-----------------|
| 文档访问404 | 链接无效或权限不足 | 提示检查文档分享权限 |
| 飞书文件格式不支持 | file/*链接无法解析 | 提示转换为docx格式 |
| 审核范围不清晰 | 需求描述不明确 | 主动询问具体审核重点 |
| 报告格式不符合要求 | 默认Markdown格式 | 提供多种格式选项或转换建议 |

| 对比维度 | 传统人工审核 | OpenClaw AI审核 |
|---------|-------------|-----------------|

| **审核速度** | 数小时/天 | 数分钟 |
