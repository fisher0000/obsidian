# OpenClaw 日报评估工作流程


### 1. 数据收集

- **来源**：指定云盘文件夹

- **操作**：列出文件夹内所有日报文档

- **读取**：逐个获取日报内容，解析关键数据

### 2. 评估标准

#### 2.1 评分维度（100分制）

| 维度       | 分值  | 评分细则                                      |
| -------- | --- | ----------------------------------------- |
| **使用频率** | 20分 | 5次以上=20分，3-5次=15分，1-2次=10分，0次=0分          |
| **使用深度** | 30分 | 复杂任务25-30分，中等15-24分，简单5-14分               |
| **业务价值** | 30分 | 节省≥2小时=25-30分，1-2小时=18-24分，30-60分钟=10-17分 |
| **创新应用** | 20分 | 重大创新16-20分，中等10-15分，一般5-9分                |

#### 2.2 评级等级

| 等级    | 分数区间    | 评级  |
| ----- | ------- | --- |
| ⭐⭐⭐⭐⭐ | 90-100分 | 优秀  |
| ⭐⭐⭐⭐  | 80-89分  | 良好  |
| ⭐⭐⭐   | 70-79分  | 中等  |
| ⭐⭐    | 60-69分  | 及格  |
| ⭐     | 0-59分   | 待改进 |

#### 3.3 日期提取规则

从文件名中提取日期（支持多种格式）：

| 格式示例         | 提取的日期      |
| ------------ | ---------- |
| `2026-03-19` | 2026-03-19 |
| `2026-03-18` | 2026-03-18 |

### 4. 评估步骤

#### 步骤1：逐一阅读所有日报


- 列出文件夹内所有日报文档

- 逐个获取并阅读每份日报的完整内容

- 记录每份日报的关键信息（暂不评分）

- 确保对所有文档有整体了解后再进入评分环节

⚠️ **禁止边读边打分的操作方式**，避免因信息不全导致评分偏差

#### 步骤2：提取日报数据

从每份日报中提取：

- 用户基本信息（姓名、部门、日期）

- 使用频率（会话次数、使用跨度、活跃时段）

- 使用深度（功能类别数、工具种类数、任务类型分布）

- 业务价值（交付成果数量、节省时间、效率提升百分比）

- 创新用法（AI识别的独特使用方式）

#### 步骤3：统一评分

在完整阅读所有日报后，进行统一评分：

- **使用频率**：直接按会话次数对应分值

- **使用深度**：根据工具多样性+任务复杂度综合判断

- **业务价值**：根据节省时间和成果数量综合判断

- **创新应用**：根据创新点的独特性和实用性判断

💡 **统一评分的好处**：

- 确保评分标准在所有文档间保持一致

- 便于横向对比，识别相对优劣

- 避免先读文档因记忆模糊而被低估

#### 步骤4：计算总分与评级

- 汇总四个维度得分

- 根据总分确定星级评级

#### 步骤5：生成评估摘要

- 列出所有用户的评分对比

- 标注优秀案例和创新亮点

- 给出整体使用趋势观察

### 5. 结果输出

#### 输出内容

1. **评估汇总表**：用户/日期/各维度得分/总分/评级

2. **亮点总结**：优秀案例、创新用法、最佳实践

3. **改进建议**：针对低分用户的提升方向

#### 输出位置

- **待定**：由任务指派者指定（多维表格/云文档/知识库等）

## 错误处理与上报机制

### 报错场景

在执行评估过程中，如遇以下情况，立即停止当前操作并在飞书群（当前群聊）中汇报：

| 错误类型     | 说明           | 上报内容           |
| -------- | ------------ | -------------- |
| **权限不足** | 无法访问日报文件夹或文档 | 文档名称/链接 + 错误提示 |
| **格式异常** | 日报内容不符合模板规范  | 文档名称 + 具体格式问题  |
| **文档损坏** | 无法读取或解析文档内容  | 文档名称/链接 + 错误信息 |
| **命名错误** | 文档命名不符合规范    | 实际文件名 + 期望格式   |
| **数据缺失** | 日报关键字段缺失     | 文档名称 + 缺失字段    |

### 上报格式

⚠️ 评估异常报告 

━━━━━━━━━━━━━━━━━━━━ 

📄 文档：[文档名称] 

🔗 链接：[文档URL] 

❌ 错误：[错误类型] 

📝 详情：[具体问题描述] 

⏰ 时间：[YYYY-MM-DD HH:MM] 

━━━━━━━━━━━━━━━━━━━━ 

### 处理流程

1. **发现错误** → 立即停止当前文档评估

2. **记录信息** → 收集文档名称、链接、错误详情

3. **群聊上报** → 按上述格式在当前群聊发送报错信息

4. **等待指示** → 等待任务指派者进一步指示后继续

## 注意事项

1. **客观性**：基于日报实际内容评分，不受主观偏好影响

2. **一致性**：相同标准适用于所有用户

3. **可追溯**：保留评估依据，便于复查

4. **保密性**：评估结果仅限授权人员查看

5. **及时上报**：遇到任何异常情况必须立即上报，不得隐瞒或跳过
