href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14 分析方法开发</a></i> 所述， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法验证</a> 是分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 的一部分。 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2（R2）</a> 对分析方法的各种 <b>验证试验</b> 的选择和评价提供指导和建议。本指导原则包括一系列术语及其定义。希望通过这些术语和定义，桥接通常存在于 ICH 成员监管机构各种药典和文件间的差异。</p><p>分析方法的验证是为了证明该分析方法适用于其预期目的。本指导原则提供了关于分析方法 <i>验证研究</i> 的进一步通用指导。</p><h2>1.2 范围</h2><p>本指导原则适用于商业化的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 和制剂（下文统称为“产品”） <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%94%BE%E8%A1%8C">放行</a> 和稳定性试验所用的分析方法。基于风险评估，该指导原则也适用于 <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> （ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10 药品质量体系</a> ）</i> 部分使用的其他分析方法。本指导原则阐明的科学原则适用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%B4%E5%BA%8A%E7%A0%94%E7%A9%B6">临床试验</a> 期间与临床阶段对应的分析方法验证。</p><p>本指导原则主要用于常用分析方法，例如 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%90%AB%E9%87%8F">含量</a> 测定、效价、纯度、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> （定量或限度检查）、鉴别或其他定量或定性测量。</p><h2>2 分析方法验证的一般考虑</h2><p>本指导原则说明了监管递交数据的要求。分析方法验证数据应按 <i><a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">《ICH M4Q 人用药品注册通用技术文档》</a></i> 相应章节要求进行递交。验证中的相关数据（以及用于计算验证结果的任何方法）均应递交，以证明该方法适用于其预期用途。如适用，可使用开发研究的数据（参见 ICH Q14）作为验证数据的一部分。当一个验证过的 <i>平台分析方法</i> 用于新目的，如果证明科学合理，验证试验可以简化。</p><p>在适当科学论证后，可接受本指导原则中未提到的其他方式进行分析方法验证。申请人负责设计最适合他们产品的验证研究和方案。</p><p>在方法学验证研究中，应使用 <i>参比物质</i> 或其他适当标定的物质，这些物质应记录鉴别、纯度或其他必要的特征。</p><p>在实际操作中，可以设计实验以便同时考虑适当的性能特征，从而得到合理且全面的分析方法性能认知，如：在 <i>可报告范围</i> 内的 <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> /选择性、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a></i> 。</p><p>如 ICH Q14 所述， <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%80%82%E7%94%A8%E6%80%A7%E8%AF%95%E9%AA%8C">系统适用性试验</a> （SST）</i> 是分析方法必要组成部分，通常在分析方法开发期间建立以作为分析方法性能的常规检查。 <i>耐用性</i> 通常在执行分析方法验证研究之前作为分析方法开发（ICH Q14）的一部分进行评价。最终，根据对分析方法的了解及其预期目的，建立 <i>分析方法验证策略</i> 。包括所需的分析方法性能，以确保测定结果的可靠（ICH Q14）。如果成功执行，分析方法验证策略将证明分析方法适用于预期目的。</p><h2>2.1 分析方法验证研究</h2><p>设计验证研究旨在提供充分证据，证明分析方法符合其目的。这些目的通过一组适当的性能特征及相关 <i>性能标准</i> 来描述，根据分析方法的预期目的和所选特定技术，性能特征及相关性能标准会有差异。第 3 节“验证试验、方法学和评价”总结了典型的方法学和验证试验供使用（验证试验的选择另见附件 1 中的图 2）。常见技术的具体非约束性示例参见附件 2。表 1（下表）列出了待测质量属性，典型性能特征和相关验证试验，进一步说明参见附件 1。</p><p>验证研究应被记录。在验证研究之前，应拟定 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81%E6%96%B9%E6%A1%88">验证方案</a> 。方案应包含关于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的预期目的、待 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81">验证</a> 的性能特征和相关标准的信息。在使用先验知识（ <b>如</b> ：来自分析方法开发阶段或既往研究知识）的情况下，应提供适当的理由。验证研究的结果应总结在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81%E6%8A%A5%E5%91%8A">验证报告</a> 中。</p><p>验证研究的实验设计应反映常规分析中生成一个 <i>可报告结果</i> 的重复测定次数。如果证明合理，可以使用不同重复测定次数进行验证测试，也可根据验证期间生成的数据调整分析方法中的重复测定次数。</p><p>图 1 显示了 ICH Q2 和 ICH Q14 之间的关联图，以及 ICH Q14 所述的分析方法开发期间产生的相关知识如何帮助设计验证研究。</p><div><table><caption>表 href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内可能会需要 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> 。这种情况下，可能需要部分或全面 <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a></i> 。科学和基于风险的原则可用于证明一个给定的性能特征是否需要再验证。再验证的程度取决于受变更影响的性能特征。</p><p>根据 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a> ，已验证的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95%E8%BD%AC%E7%A7%BB">分析方法转移</a> 属于分析方法生命周期管理范畴。将分析方法转移至不同实验室时，应进行分析方法性能特征的部分或全部 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> 和/或进行代表性样品的比较分析。如适用，应提供不进行额外转移实验的理由。</p><p><i>联合验证</i> 可通过使用在多个场所生成的数据，证明分析方法符合预定的性能标准，并且还可以满足参与场所之间分析方法转移的要求。</p><h2>2.3 可报告范围</h2><p>所需的可报告范围通常来源于质量标准并取决于方法的预期用途。通过证明分析方法提供的结果具有可接受的 <i>响应、准确度</i> 和 <i>精密度</i> ，来确认可报告范围。可报告范围应包括产品标准限度的上限和下限或报告限度（如适用）。</p><p>表 2 举例说明了常用分析方法的推荐可报告范围；如果合理，也可接受其他范围。在某些情况下，如在低浓度下，较宽的上限范围可能更切合实际。</p><h2>2.4 稳定性指示特性的证明</h2><p>如果一个经验证的定量 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 可以检测到储存期间产品相关质量属性的变化，则认为该方法具有稳定性指示性。为了证明一个稳定性指示性检测方法的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> /选择性，应将含有相关 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%99%8D%E8%A7%A3%E4%BA%A7%E7%89%A9">降解产物</a> 的样品纳入研究。这些可以包括：在样品中加入目标被分析物和已知干扰物质；暴露于各种物理和化学强制降解试验条件下的样品；以及经老化的或在强制降解试验条件下储存的实际样品。</p><h2>2.5 多变量分析方法的考虑</h2><p>对于 <i>多变量分析方法</i> ，通过使用一个以上输入变量（ <b>如：</b> 具有多个波长变量的光谱）的多变量校正模型确定结果。多变量校正模型将输入数据与关注的属性（ <b>即</b> 模型输出）的值联系起来。</p><p>多变量方法成功的验证应考虑校正、 <i>内部测试</i> 和验证。</p><p>通常分两个阶段进行开发和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81">验证</a> 。</p><ul><li>在第一阶段，模型开发包括校正和内部测试。校正数据用于创建校正模型。测试数据用于内部测试和模型优化。测试数据可以是单独的数据集或循环使用的部分 <i>校正集</i> 。内部测试步骤用于获得对模型性能的评估及微调算法的参数（如：用于偏最小二乘法（PLS） <i>潜变量</i> 的数量），在给定的数据集内选择最合适的模型。更多详细信息，请参见ICH Q14。</li></ul><ul><li>在第二阶段， <i>模型验证</i> ，使用 <i>独立样本</i> 组成的 <i>验证集</i> 对模型进行验证。对于鉴别的样品库，验证涉及对未包含在样品库中的样品（ <b>即</b> 挑战样品）进行分析，以证明样品库模型的区分能力。</li></ul><div><table><caption>表 验证试验、方法学和评价</h2><p>在以下章节中，描述了评价分析方法性能的实验方法学。根据分析方法设计所述的主要性能特征将这些方法学分组。一般公认的是，多种性能特征的信息可能来自同一数据集。如果有正当理由，可使用不同方法证明分析方法符合目的和相关性能标准。</p><h2>3.1 专属性/选择性</h2><h3>3.1.1 一般考虑</h3><p>关于分析方法的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> 或选择性，可证明无干扰，或通过对比测定结果与正交方法结果来证明。在一些情况下，专属性/选择性可能是分析方法基本科学原理的固有属性。一些实验可与准确度联合研究。</p><p>当分析方法不专属时，可证明其选择性。但是在有潜在干扰的情况下，进行被分析物鉴别或定量检测时，应尽量将干扰降至最低，并证明该分析方法适用于其预期目的。</p><p>如果一种分析方法不能提供足够区分能力，则建议采用两种或两种以上的方法予以补充，以达到所需要的专属性/选择性。</p><h4>3.1.1.1 无干扰</h4><p>专属性/选择性可通过证明被分析物的鉴别和/或定量不受共存的其他物质（如： <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%99%8D%E8%A7%A3%E4%BA%A7%E7%89%A9">降解产物</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9C%89%E5%85%B3%E7%89%A9%E8%B4%A8">有关物质</a> 、基质或可能存在的其他组分）的影响来证明。</p><h4>3.1.1.2 正交方法比较</h4><p>专属性/选择性可通过证明被分析物的测定结果与另一个经充分表征的分析方法测定结果相当来确认，理想情况下该方法应为不同测定原理的方法。</p><h4>3.1.1.3 技术固有合理性</h4><p>在某些情况下，可以通过技术参数（ <b>如：</b> 质谱中同位素的分辨率、NMR 光谱的化学位移）确保和预测分析技术的专属性。如果有适当理由，则无需进行其他额外实验研究。</p><h3>3.1.2 数据要求</h3><h4>3.1.2.1 鉴别</h4><p>对于鉴别试验，一个关键点是基于其分子结构和/或其他特性证明其能鉴别目标分析物的能力。分析方法鉴别被分析物的能力可以通过使用含有被分析物的样品得到与参比物质相当的阳性结果，以及不含被分析物的样品得到阴性结果来证明。此外，鉴别试验也应测量与待测成分结构相似或密切相关的成分的样品，以确认其不会获得阳性结果。潜在干扰成分的选择应基于科学判断，并考虑可能发生的干扰。</p><h4>3.1.2.2 含量测定、纯度和杂质检查</h4><p>应证明分析方法的专属性/选择性，满足样品中被分析物含量或效价的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 要求。</p><p>代表性数据（如：色谱图、电泳图、光谱、生物反应）应用于证明专属性，如适用，在图中标明相关成分。</p><p>对于分离技术，适当的区分性应在适宜水平考察（如：对 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%89%B2%E8%B0%B1">色谱</a> 中的关键分离，可用洗脱程度最接近的两个组份的分离度来证明其专属性）。或者，可以比较不同组分的光谱以评估干扰的可能性。</p><p>对于非分离技术（如：生物测定、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:ELISA">ELISA</a> 、qPCR），可通过使用参比物质或其他已适当标定的物质证明 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> ，以确认不存在被分析物相关的干扰。如果被分析物是工艺相关 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> ，还必须对照产品确认专属性（无干扰）。</p><p>如果认为一种方法没有专属性或没有足够的选择性，则应使用其他的方法来确保足够的区分。例如：滴定法用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 放行含量测定时，可以结合适宜的杂质检查。</p><p><u>杂质或有关物质可获得或有意生成：</u></p><p>对含量测定或效价测定，应证明被分析物在杂质和/或辅料存在时能被区分。实际操作中，可以在样品中加入适量的杂质，以证明其含量测定结果不受这些物质的影响（例如与未添加杂质的样品测定结果相比较）。或者可通过对样品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%89%A9%E6%96%99">物料</a> 进行强制降解以生成适量 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 。</p><p>对纯度或杂质检查，可以通过强制降解或在样品中添加杂质使杂质或有关物质达到适当水平以确认无干扰，证明分析方法具有区分能力。</p><p><u>杂质或有关物质不可获得：</u></p><p>如果不能制备或分离杂质、有关物质或降解产物，专属性可以通过比较含典型杂质、有关物质或降解产物的样品测定结果与正交方法测定结果来证明。采取的方法应被证明合理性。</p><h2>3.2 范围</h2><h3>3.2.1 一般考虑</h3><p>分析方法的范围是最低结果和最高结果之间的区间，在此区间内分析方法具有适当水平的响应、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 及 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> 。范围验证可通过使用适当的校正模型（即线性、非线性、多变量）对可报告结果（用于生成可报告范围）进行直接评估。在某些情况下，基于样品制备（如：稀释）和分析方法的选择，可报告范围可使用一个或多个适当的 <i>工作范围</i> 来确定。</p><p>通常，工作范围对应于呈现在分析仪器上的最低和最高样品浓度或纯度水平，在该范围内，分析方法可提供可靠的结果。生成可报告结果通常需要数学计算。可报告范围和工作范围可以相同。</p><p>如果无法获得足够纯度（或含足够量的杂质）的物质以 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81">验证</a> 整个范围（如：100% 纯度），则可适当地外推可报告范围，并应证明其合理性。</p><h3>3.2.2 响应</h3><h4>3.2.2.1 线性响应</h4><p>被分析物浓度与响应之间的线性关系应在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的整个范围内进行评估，以确认该方法适用于其预期用途。采用拟定分析方法直接验证响应，即测定产品或适当的参比物质，或分别称取的被分析物，或预先定义的各组分混匀物（如：通过已知 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%90%AB%E9%87%8F">含量</a> 的溶液稀释）。</p><p>可通过响应信号对被分析物的浓度或含量作图评价线性关系，并且应该证明分析方法在给定范围内得到测得值与真实样品值（已知的或理论的）成比例的能力。应采用适当的统计方法对实验结果进行评估（如：用最小二乘法计算回归线）。由回归线推导所得到的数据，可能有助于线性的数学估计。应提供数据图、相关系数或决定系数、y 轴截距和回归线的斜率。分析实测数据点与回归线的偏离有助于评价线性（如：对于线性响应，应评估回归分析的残差图中任何非随机模式的影响）。</p><p>验证期间评估线性，建议至少采用 5 个浓度，适当分布于整个范围。</p><p>如有必要，可对测量数据进行数学转换（例如，通过使用对数函数）。</p><p>若采用其他方法进行线性评估，应证明其合理性。</p><h4>3.2.2.2 非线性响应</h4><p>一些分析方法可能显示非线性响应。在这些情况下，需要构建一个模型或函数来描述分析方法的响应与活性/浓度之间的关系。应通过非线性回归分析（如：决定系数）评估模型的适用性。</p><p>例如，免疫测定或基于细胞的检测可能显示 S 型响应。当浓度范围足够宽时，响应受到上、下渐近线的限制，出现 S 型试验曲线。这种情况下使用的常见模型是四参数或五参数逻辑函数，不过也存在其他可接受的模型。</p><p>对于这些分析方法，线性的评估与浓度-响应曲线形状的考虑是分开的。因此，不要求浓度-响应呈线性关系。而应评估分析方法在给定范围内测得值与真实（已知或理论）样品值成比例，以评估分析方法性能。</p><h4>3.2.2.3 多变量校正</h4><p>用于构建多变量校正模型的算法可以是线性，也可以是非线性的，只要该模型适用于建立信号与关注的质量属性之间的关系。多变量方法的准确度取决于多个因素，例如校正样品在校正范围内的分布和参比分析方法误差。</p><p>在多变量分析中，测量数据通常通过导数或归一化进行预处理。</p><p>除比较参比结果和预测结果外，线性评估应包括分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AF%AF%E5%B7%AE">误差</a> （ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%AE%8B%E5%B7%AE">残差</a> ）在校正范围内如何变化的信息。残差分布图可用于评估整个工作范围内模型预测的残差。</p><h3>3.2.3 范围下限验证</h3><p>如果待测质量属性要求分析方法的范围接近该方法的范围下限， <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%A3%80%E6%B5%8B%E9%99%90%E5%BA%A6">检测限</a></i> （DL）和 <i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%AE%9A%E9%87%8F%E9%99%90%E5%BA%A6">定量限</a></i> （QL）可以使用以下方法进行估算。</p><h4>3.2.3.1 基于直观评估</h4><p>直观评价可用于非仪器分析方法和仪器分析方法。</p><p>限度的确定是通过分析已知浓度的样品及确认被分析物能被可靠分离及检测或定量的最低浓度水平。</p><h4>3.2.3.2 基于信噪比</h4><p>该方法适仅用于能呈现基线噪音的分析方法。通过比较含有已知低浓度被分析物的样品与空白样品的测量信号来测定信噪比。或者，可以使用适当基线区域的信号代替空白样品的信号。DL 或 QL 分别是被分析物被可靠检测或定量的最小浓度。一般可接受的DL 信噪比为 3:1。可接受的 QL 信噪比至少为 10:1。</p><p>信噪比的测定应在特定区域内，如可能应平均坐落于目标色谱峰的两边。</p><h4>3.2.3.3 基于线性响应值的标准偏差和标准曲线斜率</h4><p>检测限（DL）可以表示为：</p><center><i>DL</i> =3.3σ / <i>S</i></center><p>定量限（QL）可以表示为：</p><center><i>QL</i> =10σ / <i>S</i></center><p>其中 σ = 响应值的标准偏差</p><dl><dd>S = 标准曲线的斜率</dd></dl><p>斜率S 可以根据被分析物的回归线来估算。σ 可以由多种途径估算，如：</p><p><u>根据空白的标准偏差</u></p><p>通过适当数量空白样本的分析测定背景响应值的大小，计算这些响应值的标准偏差。</p><p><u>根据标准曲线</u></p><p>对含有被分析物 DL 到 QL 范围的样品进行分析测定，建立特定的标准曲线。该回归线的残差标准偏差（即均方根误差/偏差）或回归线 y-轴截距的标准偏差可作为标准偏差。</p><h4>3.2.3.4 基于范围下限的准确度和精密度</h4><p>除上述方法中描述的估算值，QL 也可以通过准确度和精密度测定直接验证。</p><h4>3.2.3.5 数据要求</h4><p>应报告 DL 及其测定方法。如果 DL 是基于直观评估或信噪比确定的，应提供相关数据作为可接受依据。</p><p>如通过计算或外推法获得 DL 的估算值，应通过单独分析适当份数接近或等于 DL 浓度的样品来验证该估算值。</p><p>应报告 QL 及其测定方法。</p><p>如果 QL 是估算值，应通过分析适当份数接近或等于定量限浓度的样品来验证该估算值。如果 QL 远低于报告限（例如，比报告限低约 10 倍），则可免除前述确认验证，并提供相关理由。</p><p>对于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 检查，分析方法的定量限应等于或低于报告阈值。</p><h2>3.3 准确度和精密度</h2><p>准确度和精密度可以单独评估，各自拟定预先设定的标准。或者，也可以合并评价准确度和精密度。</p><h3>3.3.1 准确度</h3><p>应在分析方法的可报告范围内评估准确度，通常通过比较测定结果与期望值进行验证。应在分析方法的常规检测条件下验证准确度（如：对含有基质的样品，采用方法所述样品制备步骤操作）。</p><p>通常，准确度可通过下述研究之一来验证。在某些情况下，如果精密度、范围内响应和专属性已经确证，方法准确度可以通过推论得出。</p><h4>3.3.1.1 与标准物质比较</h4><p>分析方法测定已知纯度的被分析物（如：参比物质、经过充分表征的杂质或有关物质），并将测定结果 <i>和</i> 理论期望结果进行比较。</p><h4>3.3.1.2 加标研究</h4><p>分析方法测定加标样品，该样品是在含有除被分析物外其他所有成分的基质中添加了已知量的目标分析物。如无法获得基质全部组分，可将被分析物加入到或富集在检测样品中进行测定。通过比较未加标样品和加标/富集样品的测定结果进行评估。</p><h4>3.3.1.3 与正交方法比较</h4><p>将拟定分析方法的测定结果与正交分析方法的测定结果进行比较。应报告正交分析方法的准确度。在无法获得模拟加标研究所需的所有相关成分的情况下，正交方法可与定量杂质测定一起用于确认主要的测量值。</p><h4>3.3.1.4 数据要求</h4><p>准确度需要在可报告范围内，使用各浓度水平适当数量的 <i>平行样品</i> 进行评估（如： 按完整分析方法在 3 个浓度各重复测定 3 次）。</p><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 应报告为样品中添加已知量的被分析物的平均回收率，或平均值与可接受真值的差值及适当的 100（1 - α）% <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%BD%AE%E4%BF%A1%E5%8C%BA%E9%97%B4">置信区间</a> （或其他合理的统计区间）。除另有说明外，观察到的区间应与相应的准确度可接受标准相匹配。</p><p>对于杂质检查，应描述测定单个杂质或杂质总量的方法（如：相对于主成分的重量百分比或面积百分比）。</p><p>对于多变量分析方法的定量应用，应使用适当的指标，如：应使用均方根预测误差（RMSEP）。如果发现 RMSEP 与可接受的均方根校正误差（RMSEC）相当，则表明使用独立检测集进行检测时，该模型足够准确。定性应用如分类、误分类率或阳性预测率可用于表征准确度。</p><h3>3.3.2 精密度</h3><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%90%AB%E9%87%8F">含量</a> 测定和杂质（纯度）定量测定的方法验证，应涵盖 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> 研究。</p><p>精密度考察应使用真实均质的样品，或当上述样品不可获得时，可使用人为配制的样品（如：在基质混合物中加标或在样品中进行被分析物富集）。</p><h4>3.3.2.1 重复性</h4><p><i>重复性</i> 的验证方法：</p><dl><dd>a) 涵盖方法可报告范围内，至少测定 9 次（如：3 个浓度/每个浓度重复测定 3 次）</dd></dl><dl><dd>或</dd></dl><dl><dd>b) 100%的试验浓度至少测定 6 次。</dd></dl><h4>3.3.2.2 中间精密度</h4><p><i>中间精密度</i> 的验证程度应根据分析方法的预期使用情况而定。申请人应确定随机事件对分析方法精密度的影响。需研究的典型变动因素包括不同日期、环境条件、分析者和仪器设备，如相关。理想情况下，考察的变量应该基于对分析方法开发和风险评估（ICH Q14）时的理解并予以证明。无需单独研究这些影响。鼓励使用 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:DOE">试验设计</a> （DoE）研究。</p><h4>3.3.2.3 重现性</h4><p>通过不同实验室协同检验评价 <i>重现性</i> 。申请递交时通常不需要考察重现性，但当分析方法需要标准化时，例如收载到药典中，以及在多个场地使用该分析方法的情况，则应考虑进行重现性研究。</p><h4>3.3.2.4 数据要求</h4><p>应报告标准偏差、相对标准 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%81%8F%E5%B7%AE">偏差</a> （变异系数）和适当的 100（1 - α）% <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%BD%AE%E4%BF%A1%E5%8C%BA%E9%97%B4">置信区间</a> （或其他合理的统计区间）。除另有说明外，观察到的区间应与对应的精密度可接受标准应相匹配。</p><p>此外，对于多变量分析方法，RMSEP 的常规指标包括准确度和精密度。</p><h3>3.3.3 准确度和精密度的合并评价方式</h3><p>替代单独评估 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> 的方法是考虑二者的总体影响，通过一个合并的性能标准进行评估。</p><p>方法开发期间生成的数据可能有助于确定最佳方法，并确定适当的性能标准，用于合并的准确度和精密度的评估。</p><p>可使用预测区间、容许区间或置信区间来评价合并的准确度和精密度。如果合理，其他方法也可以接受。</p><h4>3.3.3.1 数据要求</h4><p>如果选择合并的性能标准，结果应报告为一个综合值，以提供分析方法适用性的适当总体认知。如果与证明分析方法适用性相关，应提供准确度和精密度的单独结果作为补充信息。应描述使用的方法。</p><h2>3.4 耐用性</h2><p>在方法开发阶段，根据所研究分析方法的类型，对分析方法在预期操作环境中的适用性进行评价。耐用性试验是通过对 <i>分析方法参数</i> 进行有意地改变后，确定分析方法的可靠性。如适用，应考察分析方法实施期间样品制备液和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AF%95%E5%89%82">试剂</a> 的稳定性。耐用性评价可根据具体情况作为分析方法开发数据的一部分提交，或应根据要求提供。</p><p>更多详细信息，请参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a> 。</p><h2>4 术语</h2><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> （ACCURACY）</b></p><p>分析方法的准确度指的是常规的真实值或认可的参比值与测定值或测定值集之间的相近程度。 <i>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> (ANALYTICAL PROCEDURE)</b></p><p>分析方法是指进行分析操作的方式。分析方法应足够详细地描述每个分析检测所必需的步骤。 <i>（ICH Q2）</i></p><p><b>分析方法参数 (ANALYTICAL PROCEDURE PARAMETER)</b></p><p>分析因素（包括 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AF%95%E5%89%82">试剂</a> 级别）或分析方法的操作条件，这些操作条件可连续变化（如，流速）或规定在可控的单一水平。 <i>（ICH Q14）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法验证</a> 策略 (ANALYTICAL PROCEDURE VALIDATION STRATEGY)</b></p><p>分析方法验证策略描述用于验证的分析方法性能特征的选择。在策略中，开发研究期间收集的数据和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%80%82%E7%94%A8%E6%80%A7%E8%AF%95%E9%AA%8C">系统适用性试验</a> （SST）可用于验证，并且预先设定一组适当的验证试验。 <i>（ICH Q14）</i></p><p><b>校正模型 (CALIBRATION MODEL)</b></p><p>一种基于已知样品分析检测的模型，该模型将输入数据与关注属性值（ <b>即</b> 模型输出）相关联。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> (CONTROL STRATEGY)</b></p><p>根据当前对产品和工艺的理解而产生的一系列保证工艺性能和产品质量的有计划的控制。这些控制可包括与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 和制剂的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%89%A9%E6%96%99">物料</a> 与组分、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%82%E6%88%BF">厂房</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E5%A4%87">设备</a> 运行条件、过程控制、成品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 相关的参数和特性，以及相应的监控方法和频次。 <i>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）</i></p><p><b>联合验证 (CO-VALIDATION)</b></p><p>证明分析方法在不同实验室用于相同预期目的时，其符合预设的性能标准。由于实验室不同而受到潜在影响，联合 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81">验证</a> 可能包含全部（完全 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> ）或部分（部分再验证）性能特征。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%A3%80%E6%B5%8B%E9%99%90%E5%BA%A6">检测限</a> （DETECTION LIMIT, DL）</b></p><p>检测限是指样品中的被分析物能够被检测但无需定量为确切值的最低量。 <i>（ICH Q2）</i></p><p><b>测定 (DETERMINATION)</b></p><p>根据验证方案，对一个配制的样品进行单次或重复测定得到的报告值。 <i>（ICH Q2）</i></p><p><b>中间 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> (INTERMEDIATE PRECISION)</b></p><p>中间精密度是指实验室内变异性。考虑的因素应包括变异性潜在的来源，例如不同天、不同环境条件、不同分析人员和不同仪器。 <i>（ICH Q2）</i></p><p><b>性能特征 (PERFORMANCE CHARACTERISTIC)</b></p><p>确保测定结果可靠的一种特征描述，该特征与分析技术无关。通常可能包括 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> /选择性和范围。既往ICH Q2 版本称之为论证项目。 <i>（ICH Q2）</i></p><p><b>性能标准 (PERFORMANCE CRITERION)</b></p><p>描述一个给定性能特征的数值范围、限度或期望状态的可接受标准，以确保检测结果的可靠。 <i>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a> ）</i></p><p><b>平台分析方法 (PLATFORM ANALYTICAL PROCEDURE)</b></p><p>适用于测定不同产品的质量属性而不需要对其操作条件、系统适用性和报告结构进行重大改变的分析方法。这类分析方法可用于分析属性非常相似的分子，相关属性即为平台分析方法所测定。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> (PRECISION)</b></p><p>分析方法的精密度是指在规定条件下，对均质样品多次取样检测得到一系列检测结果之间的接近程度（离散程度）。精密度可以从三个层面考虑：重复性、中间精密度和重现性。</p><p>分析方法的精密度通常以一系列多次检测结果的方差、标准 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%81%8F%E5%B7%AE">偏差</a> 或变异系数来表示。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%AE%9A%E9%87%8F%E9%99%90%E5%BA%A6">定量限</a> （QUANTITATION LIMIT, QL）</b></p><p>定量限是指在有适当的准确度和精密度，能够定量测定样本中被分析物的最低量。定量限是对样品基质中低含量物质进行定量测定时需要考察的参数，特别适用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 和/或 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%99%8D%E8%A7%A3%E4%BA%A7%E7%89%A9">降解产物</a> 的测定。 <i>（ICH Q2）</i></p><p><b>范围 (RANGE)</b></p><p>分析方法的范围是最低结果和最高结果之间的区间，在此区间内分析方法具有适当水平的精密度、准确度及响应。 <i>（ICH Q2）</i></p><p><b>可报告范围 (REPORTABLE RANGE)</b></p><p>分析方法的可报告范围包括从最低至最高可报告结果的所有值，这些值具有适当水平的精密度和准确度。可报告范围的单位通常与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 中的可接受标准相同。 <i>（ICH Q2）</i></p><p><b>工作范围 (WORKING RANGE)</b></p><p>工作范围对应于能在分析仪器上的待测质量属性（例如含量或纯度）的最低和最高水平，且在此范围内分析方法提供可靠结果。 <i>（ICH Q2）</i></p><p><b>参比物质 (REFERENCE MATERIAL)</b></p><p>一种经过适当标定的物质，在一个或多个定义的属性方面具有足够均匀性和稳定性，且确定适用于预期目的。参比物质可能包括国家/国际标准物质、药典标准物质、内部一级/二级参比物质。 <i>（ICH Q2）</i></p><p><b>重复性(REPEATABILITY)</b></p><p>重复性是指在同样操作条件下，短时间间隔内测量的精密度。重复性也称为测量精密度（Intra-assay precision）。 <i>（ICH Q2）</i></p><p><b>可报告结果(REPORTABLE RESULT)</b></p><p>经计算或处理，并考虑所述重复性测量后，由分析方法得到结果 <i>（ICH Q2）</i></p><p><b>重现性 (REPRODUCIBILITY)</b></p><p>重现性是指不同实验室之间测量的精密度（ <b>如</b> ：实验室间研究，通常用于方法的标准化）。 <i>（ICH Q2）</i></p><p><b>响应 (RESPONSE)</b></p><p>分析方法的响应是分析方法（在给定范围内）获得信号的能力，通过某些已知的数学函数能将这些获得的信号与样品中被分析物浓度（含量）或活性有效联系起来。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> (REVALIDATION)</b></p><p>证明产品、工艺或分析方法本身变更后，分析方法仍符合其预期目的。 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> 可涉及所有（完整再验证）或部分（部分再验证）的性能特征。 <i>（ICH Q2）</i></p><p><b>耐用性 (ROBUSTNESS)</b></p><p>分析方法的耐用性是衡量其在正常使用时符合预期性能标准的能力。通过有意变化分析方法参数来检验耐用性。 <i>（ICH Q14）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> /选择性 (SPECIFICITY/SELECTIVITY)</b></p><p>在用指定分析方法测定某一分析物时，专属性和选择性均用于描述其他物质的干扰程度。专属性通常用于描述最终状态，即分析方法可以明确检测目标分析物。选择性则是一个相对术语，用于描述混合物或基质中特定被分析物可被检测且不受具有类似行为的其他组分干扰的程度。 <i>（ICH Q2）</i></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%80%82%E7%94%A8%E6%80%A7%E8%AF%95%E9%AA%8C">系统适用性试验</a> （SYSTEM SUITABILITY TEST, SST）</b></p><p>开发系统适用性试验用于确认检测系统及与分析方法相关的分析操作适用于预期目的，并提高对于分析方法性能不可接受情况的可检测性。 <i>（ICH Q14）</i></p><p><b>验证研究 (VALIDATION STUDY)</b></p><p>对先验知识、数据或专项实验（ <b>即</b> 验证试验）的评估，以确定分析方法适用其预期目的。 <i>（ICH Q2）</i></p><p><b>验证试验 (VALIDATION TEST)</b></p><p>验证试验是为验证分析方法适用其预期目的而设计的专项实验。 <i>（ICH Q2）</i></p><p><u><b>多变量术语 (MULTIVARIATE GLOSSARY)</b></u></p><p><b>校正集 (CALIBRATION SET)</b></p><p>一组已知特性和分析检测结果相匹配的数据。 <i>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a> ）</i></p><p><b>独立样本 (INDEPENDENT SAMPLE)</b></p><p>独立样本是指未包含在多变量模型校正集中的样品。独立样本与校正样本可来源于同一批次。 <i>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）</i></p><p><b>内部测试 (INTERNAL TESTING)</b></p><p>内部测试是检查特定样品数据经过模型处理后预测是否正确（定性或定量）的过程。</p><p>内部测试用于确定潜变量的最佳数量、估算标准误差及检测潜在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%A6%BB%E7%BE%A4%E5%80%BC">离群值</a> 。 <i>（ICH Q2）</i></p><p><b>潜变量 (LATENT VARIABLES)</b></p><p>与测量变量直接相关经由数学推导而得的变量，潜变量用于进一步分析处理。 <i>（ICH Q2）</i></p><p><b>模型验证 (MODEL VALIDATION)</b></p><p>通过使用独立的试验数据挑战模型，并将结果与预定性能标准进行比较，从而确定模型适用性的过程。 <i>（ICH Q2）</i></p><p><b>多变量 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> (MULTIVARIATE ANALYTICAL PROCEDURE)</b></p><p>通过使用多于一个输入变量的多变量校正模型而确定结果的分析方法。 <i>（ICH Q2）</i></p><p><b>参比分析方法 (REFERENCE ANALYTICAL PROCEDURE)</b></p><p>单独的分析方法，用于获得多变量分析方法中校正样本和验证样品的参比值。 <i>（ICH Q2）</i></p><p><b>验证集 (VALIDATION SET)</b></p><p>用于对校正模型的性能进行独立评估的一组数据。 <i>（ICH Q2）</i></p><h2>5 参考文献</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10 药品质量体系</a></p><p><a
