日</td></tr></tbody></table></div><h2>1. 简介</h2><p>本指导原则描述了原料药工艺开发及对工艺理解的方法，也为 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">通用技术文档</a> （CTD）模块 3 中的 3.2.S.2.2 部分至3.2.S.2.6 部分（ <a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH-M4Q</a> ）应当提供哪些信息提供指导⁠⁠⁠⁠⁠⁠。该指导原则侧重于关于原料药开发和生产方面的问题，包括为降低杂质而设计的步骤。另外，ICH Q11 还对 ICH 指导原则药品开发（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">Q8</a> ）、质量风险管理（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q9_R1_Quality_Risk_Management_20230118">Q9</a> ）和药品质量体系（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">Q10</a> ）中关于原料药开发和生产方面的原则和概念进行了进一步的阐释。</p><p>制药公司在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 开发时可以选择采用不同的方法。出于阐明该指导原则的目的，术语“传统的”（traditional）和“增强的”（enhanced）用于区分两种不同的开发方式。在传统方式中， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%8F%82%E6%95%B0">工艺参数</a> 的设定点及操作范围是确定的，原料药的控制策略典型的依赖于工艺的可重复性和终产品检验符合已建立的质量标准。在增强的方式中，风险管理和科学知识被更加广泛地用于确定和理解影响 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> （CQAs）的工艺参数及 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8D%95%E5%85%83%E6%93%8D%E4%BD%9C">单元操作</a> ，并且开发一些合适的控制策略应用于原料药开发的整个生命周期，包括设计空间的建立。正如在 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> 中对药物制剂所讨论的，充分理解原料药及其生产工艺，可以为更加灵活的监管方式的建立奠定基础。监管的灵活程度通常取决于在上市许可的申报资料中所提供的相关科学知识的水平。</p><p>传统方式和增强方式并不互相排斥。制药公司在原料药的开发中既可使用传统的方式，也可使用增强的方式，或同时使用两种方式。</p><h2>2. 范围</h2><p>本指导原则适用于 ICH 指导原则 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6A_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_New_Drug_Substances_and_New_Drug_Products:_Chemical_Substances_199910">Q6A</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">Q6B</a> 的范围章节所定义的原料药，也可在咨询监管部门后用于其他类型的产品。其主要涉及通用技术文档（ <a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH M4Q</a> ）模块 3 的第3.2.S.2.2 部分到第 3.2.S.2.6 部分内容的准备和组织。本指导原则不适用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%8D%AF%E5%93%81%E7%A0%94%E5%8F%91">药品开发</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%B4%E5%BA%8A%E7%A0%94%E7%A9%B6">临床研究</a> 阶段的申报内容。然而，本指导原则中的开发原则是在临床研究阶段应考虑的重点内容。</p><p>本指导原则并不涵盖上市后变更的地区性要求。</p><h2>3. 生产工艺开发</h2><h2>3.1 总则</h2><p>原料药生产工艺开发的目的是建立一个能够持续生产出预期质量原料药的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%95%86%E4%B8%9A%E5%8C%96%E5%A4%A7%E7%94%9F%E4%BA%A7">商业化生产</a> 工艺。</p><h3>3.1.1 与药物制剂相关的原料药质量属性</h3><p>原料药预期质量的确立，应考虑到原料药在制剂中的使用情况，同时应对其 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%90%86%E5%8C%96%E7%89%B9%E6%80%A7">理化性质</a> 、生物和微生物属性或特性有充分的认识和理解，这将对药物制剂的开发产生影响（例如，原料药的溶解性可能会影响剂型的选择）。 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:QTPP">目标产品质量概况</a> （QTPP）、药物制剂潜在的关键质量属性（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> 中的定义）及之前相关产品的经验有助于确认原料药潜在的关键质量属性。对于关键质量属性的认识和理解可以在开发的过程中不断深入。</p><h3>3.1.2 工艺开发工具</h3><p>质量风险管理（QRM，参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q9_R1_Quality_Risk_Management_20230118">ICH Q9</a> 中的表述）可以被用于多种活动中，包括评估生产工艺设计的选择，评估质量属性和生产 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%8F%82%E6%95%B0">工艺参数</a> ，以及增加日常生产批次的预期质量保证度。可以在工艺开发的早期开展风险评估，并在获得更多知识和更深理解时重复使用。可以使用正式的或非正式的风险管理工具，如公认的工具或内部程序。</p><p>知识管理（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> 中的表述）也有助于生产工艺的开发。在本指导原则中，潜在的信息来源包括先前的知识和开发研究。先前的知识包括已经建立的生物学、化学和工程学原理、技术文献，及已经应用的生产经验。来源于相关的先前知识的数据，包括平台制造（参见术语），可以用来支持商业工艺开发和加深对科学的理解。</p><h3>3.1.3 开发方法</h3><p><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> 提出“药品的开发策略因公司而异，也因产品而异，药品开发的方法和范围也各不相同，这些信息均应在申报资料中予以介绍”。这些理念同样适用于原料药生产工艺的开发。申请人在原料药的开发中可以选择采用传统的方式或增强的方式，或同时使用两种方式。</p><p>生产工艺开发至少应当包括以下要素：</p><ul><li>确定与原料药相关的潜在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> ，以便于研究和控制这些对药物制剂质量产生影响的属性；</li></ul><ul><li>确定一个合适的生产工艺；</li></ul><ul><li>确定一个控制策略，以确保工艺的实施和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 质量。采用增强的方式进行生产工艺开发还需要额外包括以下要素：</li></ul><ul><li>评估、理解和改进生产工艺的系统方法，包括：</li></ul><dl><dd>○通过先前的知识、实验和风险评估来确认可能对原料药的关键质量属性产生影响的物料属性（例如，原材料、起始原料、试剂、溶剂、工艺助剂、中间体的属性）及工艺参数；</dd></dl><dl><dd>○确定物料属性和工艺参数与原料药关键质量属性相关联的功能性关系。</dd></dl><ul><li>采用增强方式结合质量风险管理（QRM）来建立一个合适的控制策略，例如，其中可以包括对设计空间的建议。</li></ul><p>通过采用增强方式获得的更多知识和更深入的理解，可以促进产品在整个 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 的持续改进和创新（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）。</p><h3>3.1.4 原料药的关键质量属性</h3><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> （CQAs）是一个物理、化学、生物学或微生物学属性或特征，其应该被限定在一个合适的限度、范围或分布内，以确保所需的产品质量。潜在的原料药关键质量属性被用于指导工艺开发。随着对原料药知识和工艺的理解不断加深，潜在的关键质量属性目录可以随之被修改。</p><p>原料药的关键质量属性通常包括那些影响鉴别、纯度、生物活性和稳定性的属性或特征。当物理性质对药物制剂的生产或性能产生重要影响时，也可将其指定为关键质量属性。对于生物技术产品/ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E7%89%A9%E5%88%B6%E5%93%81">生物制品</a> 而言，制剂的大部分关键质量属性与原料药相关，因此原料药的设计或者生产工艺直接决定了生物技术产品/生物制品的关键质量属性。</p><p>由于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 可能会对药物制剂的安全性产生潜在的影响，因此杂质是原料药一类重要的关键质量属性。对化学实体而言，杂质可以包括有机杂质（包括潜在的致突变杂质）、无机杂质，例如金属残留物、以及残留溶剂（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q3A(R2)_Impurities_in_New_Drug_Substances">ICH Q3A</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q3C_R9_Impurities_Guideline_for_Residual_Solvents_20240124">Q3C</a> ）。对生物技术产品/生物制品而言，杂质可能与工艺相关或与产品相关（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">ICH Q6B</a> ）。与工艺相关的杂质包括：源于细胞基质的杂质（例如： <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%AE%BF%E4%B8%BB%E8%9B%8B%E7%99%BD">宿主细胞蛋白</a> （HCP）和 DNA）；源于细胞培养的杂质（例如：培养基组分），及源于下游工艺的杂质（例如：柱滤出物）。确认生物技术产品/生物制品的关键质量属性时，也应考虑包括 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">Q6B</a> 中规定的污染物，包括所有偶然因素引入的、与生产工艺无关的物质（例如：外来病毒、细菌或支原体污染物）。</p><p>确认复杂产品的关键质量属性具有挑战性。例如，生物技术产品/生物制品通常具有大量的质量属性，不可能逐一全面评价其对产品安全性和有效性的影响。可以用风险评估的方法对质量属性排序或确定优先级。先前的知识可以用在原料药开发的初始阶段，并根据 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 中的开发数据（包括来自非临床和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%B4%E5%BA%8A%E7%A0%94%E7%A9%B6">临床研究</a> 数据）持续更新评估。关于作用机制和生物学特性的知识，例如评价构效关系的研究，可用于某些产品属性的风险评估。</p><h3>3.1.5 物料属性和工艺参数与原料药关键质量属性的关联</h3><p>生产工艺的开发程序应当确认哪些物料的属性（例如原材料、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B5%B7%E5%A7%8B%E7%89%A9%E6%96%99">起始原料</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AF%95%E5%89%82">试剂</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%BA%B6%E5%89%82">溶剂</a> 、工艺助剂、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E9%97%B4%E4%BA%A7%E5%93%81">中间体</a> 的属性） 和工艺参数应该被控制。风险评估可以帮助确认对原料药的关键质量属性有潜在影响的物料属性和工艺参数。对原料药质量有重要影响物料属性和工艺参数应采用控制策略进行控制。</p><p>帮助确定与原料药上游物料相关的控制策略要素的风险评估可以包括对生产工艺能力、属性检测能力，以及对原料药质量影响严重程度的评估。例如，当评估原材料或中间体的某个杂质与原料药关键质量属性的关联时，应考虑原料药生产工艺去除该杂质或其衍生物的能力。与杂质相关的风险通常可以通过原材料/中间体的质量标准，和/或下游步骤中稳健的纯化能力来进行控制。对于那些在原料药检测能力上有内在限制的关键质量属性（如，病毒安全性）同样可以用风险评估进行确认。在这种情况下，这些关键质量属性应当在生产工艺上游的合适步骤进行控制。</p><p>对于化学实体药物的开发，主要关注对于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 的了解和控制。理解杂质的形成、转化（杂质是否发生反应和发生结构的转变）和清除（杂质是否能够通过重结晶，萃取等操作除去）以及与原料药关键质量属性的最终杂质之间的关系非常重要。由于杂质通过多个工艺步骤产生，因此应当通过评价工艺来建立合适的杂质控制策略。</p><p>采用传统方式，物料的质量标准和工艺参数范围主要基于批工艺历史和单变量实验。增强方式可以更全面地理解物料属性和工艺参数与关键质量属性的关系和相互作用的影响。案例 1（参见第 10.1 部分）说明了如何运用先前的知识和化学第一原则来进行工艺参数的开发。</p><p>开发过程中，可以使用风险评估来确认那些可能影响潜在关键质量属性的工艺步骤。进一步的风险评估可用于对工艺与质量关系需要有更好理解的开发工作。如采用增强方式，确定合适的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%89%A9%E6%96%99">物料</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 及工艺参数范围需要遵循以下步骤：</p><ul><li>确认工艺变化的潜在源头；</li></ul><ul><li>确认可能会对原料药质量产生最大影响的物料属性及工艺参数。这可以基于之前的知识和风险评估工具；</li></ul><ul><li>设计并进行研究（例如，机理和/或动力学评价、多变量试验设计、模拟、建模）来识别与确定物料属性和工艺参数与原料药关键质量属性之间的关联和关系；</li></ul><ul><li>对数据进行分析与评估，来设定合适的范围，包括必要时建立的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E8%AE%A1%E7%A9%BA%E9%97%B4">设计空间</a> 。</li></ul><p>可以开发小规模的模型，并用于支持工艺开发研究。模型开发时应考虑规模效应，及对所拟定商业化生产工艺的代表性。一个科学制定的模型能够预测质量，并可以被用于支持多种规模和设备的操作条件的外推研究。</p><h3>3.1.6 设计空间</h3><p>设计空间是输入变量（例如，物料属性）和已被证明能够提供质量保证的工艺参数的多维组合和相互作用。在设计空间范围内的活动不被视为变更。超出设计空间的活动被视为变更，通常需要启动监管部门批准该变更。设计空间由 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 提出，并由监管部门评估与批准（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> ）。</p><p><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> 采用增强方式针对药物制剂开发进行的设计空间方面的考虑内容同样适用于原料药。准确评估物料属性和工艺参数变动对原料药关键质量属性的重要性和影响进而确定设计空间的界限的能力，取决于对工艺及产品的理解程度。</p><p>可以在先前知识、基本原则、和/或对工艺的经验性理解的基础上进行设计空间的开发。可以使用模型（例如，定性的、定量的模型）来支持多种规模和设备的设计空间。</p><p>可以确定每个单元操作（例如，反应、结晶、蒸馏、精制）的设计空间，或者确定选用的单元操作的组合的设计空间。包含这样设计空间的单元操作的选择通常基于它们对关键质量属性的影响，且不必是连续的。应对工艺步骤之间的关联性进行评估，例如，以便于控制杂质的累积产生和去除。一个跨多个 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8D%95%E5%85%83%E6%93%8D%E4%BD%9C">单元操作</a> 的设计空间可以提供更大的操作灵活性。</p><p>对于一些生物技术/生物原料药，由于各种因素，包括工艺的变异性和原料药的复杂性（例如：翻译后修饰），开发和批准的设计空间可能会面临挑战。这些因素可能会影响到残留风险（例如，基于与规模敏感性相关的不确定性对关键质量属性产生潜在的不可预测的改变)，在设计空间被批准后，这些风险仍然存在。根据残留风险的水平，申请人最好能提供批准后如何对设计空间内的改变进行管理的方案，这些方案应指出如何利用工艺知识、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 及特性方法评估在已批准的设计空间内的变化对产品质量的影响。</p><h2>3.2 生产工艺开发信息的递交</h2><p>原料药生产工艺开发部分提供的信息（主要在申请申报的第 3.2.S.2.6 部分）应当指明工艺开发过程中的重要变更，将相关原料药批次与用于制备这些样品的生产工艺的开发阶段相关联，并解释如何运用先前的知识、风险评估和其他研究（例如，实验、建模、模拟）来建立生产工艺和控制策略的重要方面。工艺开发的信息应当有逻辑的组织起来，以易于理解。生产企业可以用多种不同的方法呈现工艺开发信息，以下建议可供考虑。</p><h3>3.2.1 工艺开发方面的总结</h3><p>推荐在生产工艺开发部分的开始，用一个叙述性地概要对工艺开发过程中的重要里程碑进行描述，并解释它们是如何链接起来以确保获得预期原料药的质量。该概要中应包括以下内容：</p><ul><li>原料药关键质量属性目录；</li></ul><ul><li>生产工艺的演变阶段及控制策略的相关变更方面的简述；</li></ul><ul><li>确认对原料药的关键质量属性产生影响的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%89%A9%E6%96%99">物料</a> 属性及 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%8F%82%E6%95%B0">工艺参数</a> 的简述；</li></ul><ul><li>任何设计空间开发方面的简述。</li></ul><p>在工艺开发总结之后，接下来的生产工艺开发部分应包括更详细的信息，如下面所推荐的内容。</p><h3>3.2.2 原料药的关键质量属性</h3><p>应当列出原料药的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> （CQAs），并给出设定这些性质或特性作为关键质量属性的理由。在某些情况下，可能需要适当的解释为什么可能被视为潜在 CQA 的其他属性或特征未包含在 CQA 列表中。应当在申报资料中的其他部分（例如，3.2.S.3.1 结构解析和其他特性）提供信息的链接信息或参考文献，来支持设定这些性质或特性作为关键质量属性。某些与药物制剂关键质量属性相关的原料药关键质量属性的讨论可适当体现在申报资料的药物开发部分（例如，3.2.P.2.1 药物制剂中的组分）。</p><h3>3.2.3 生产工艺的历史</h3><p>应当描述并讨论生产工艺的重要变更或者是用于支持上市申请（例如，用于支持上市申请的非临床研究或临床研究或稳定性研究）的原料药批次和生产规模批次制备地址的变更。通常按照时间顺序描述，从开始直至拟定的商业化生产工艺为止。应该提供批信息（批量或规模、生产地点与日期、使用的路线与工艺，以及用途[例如，用于特定的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%AF%92%E7%90%86%E5%AD%A6">毒理学</a> 研究或临床研究]）以及从相关原料药批次的对比分析中获得的支持性数据，或者提供参考的资料（例如，批分析，3.2.S.4.4部分）。</p><p>对于生物技术/生物制品原料药，每个重大变更的原因都应当进行解释，并且评估该变更对原料药（和/或中间体，适当条件下）质量的影响。生产工艺历史部分应包括 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5E_Comparability_of_Biotechnological/Biological_Products_Subject_to_Changes_in_their_Manufacturing_Process">ICH-Q5E</a> 中描述的对研发中的可比性进行讨论。应包括测试选择依据及结果评估的讨论资料。用于评估生产变更对原料药及相应药物制剂影响的测试实验也可以包括非临床研究和临床研究。应指出这些研究在申报资料的其他模块间交叉引用的位置。</p><h3>3.2.4 工艺开发研究</h3><p>在申报资料中引用的、用于确立商业化生产工艺重要特征和控制策略的研究和风险评估应该被列出（例如，用表格的形式）。应提供每个被引用的研究或风险评估的目的或最终用途。</p><p>每个引用的研究或风险评估都应当有充分详细的总结，以便于理解研究的目的、收集的数据、如何进行分析、得到的结论以及研究对生产工艺或生产工艺的进一步开发的影响。应该结合商业化生产工艺（参见 3.2.S.2.2 部分所述）中的目标操作条件或设计空间，对其中的特定参数及其范围的研究进行描述和讨论。应该对确立设计空间所用的风险评估工具和研究结果进行充分描述。案例 2（参见第 10.2 部分） 中展示了在设计空间的发展过程中，对评价参数进行风险排序的一种沟通工具。在开发时如果涉及先前的知识时，应提供相关的信息和资料，在适当情况下，应该对特定原料药的相关性进行论证。</p><p>应该对用于支持商业化工艺开发的小规模模型的情况进行叙述。</p><h2>4. 生产工艺与过程控制的描述</h2><p>原料药生产工艺的描述代表了申请人对原料药生产做出的承诺。应该充分地描述生产工艺和过程控制相关信息（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH M4Q</a> 的 3.2.S.2.2 部分）。</p><p>应当采用流程图和顺序叙述的形式提供生产工艺的描述。在描述中应指出工艺过程的每一步或每一阶段的过程控制点。当工艺步骤具有规模效应时，对于跨越多个操作规模的生产步骤应包括规模参数。生产工艺的任何设计空间都应当作为生产工艺的一部分来描述。案例 （3 参见第 10.3 部分）列出了一个生物技术产品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E8%AE%A1%E7%A9%BA%E9%97%B4">设计空间</a> 的实例。</p><p>很多生物技术/生物制品具有复杂的上游工艺，使用分割与合并的方法来产生一个原料药批次。生产商应该对如何定义原料药的批次进行解释说明（例如，收获物或中间体的分割与合并)。应当包含批量或规模及批号的细节。</p><h2>5. 起始原料和源物质的选择</h2><h2>5.1 总则</h2><h3>5.1.1 化学合成原料药的起始原料选择</h3><p>在决定原料药生产工艺的起点时，应当考虑下列原则（即，起始原料的选择）。</p><ul><li>通常，物料特性或操作条件的改变越靠近生产工艺的初始阶段，则影响原料药质量的可能性就越小；</li></ul><dl><dd>○风险与生产工艺的步骤数之间的关系是两个因素所决定的，一个与原料药的物理属性有关，另一个与杂质的形成、转化和清除有关。原料药的物理属性取决于最后的结晶步骤及后续操作（例如，研磨、微粉化），这些都在生产工艺的末期发生。生产工艺早期引入或产生的杂质通常比生产工艺后期生成的杂质有更多的机会在纯化操作中被除去（例如洗涤、已分离中间体的结晶），因此较少可能被带入 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 中。然而，有时候（例如，用固体载体合成多肽或聚核苷酸），风险与生产工艺的步骤数之间的关系也非常有限。</dd></dl><ul><li>监管部门评估对原料药及原料药生产工艺的控制是否进行了充分考虑，包括是否对杂质进行了适当的控制。为开展该项评估，应在申报资料中充分描述原料药的生产工艺，以便监管部门了解杂质在工艺过程中是如何形成的、工艺如何变更会对杂质的形成、去向和去除产生影响、提议的控制策略为什么适合于原料药的生产工艺。这通常将包括对多个化学转化步骤的描述；</li></ul><ul><li>申报资料 3.2.S.2.2 部分的生产工艺描述中通常应该包含对原料药的杂质谱产生影响的生产步骤；</li></ul><ul><li>如果原料药的生产工艺有多个支链，则每个支链追溯至其使用 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B5%B7%E5%A7%8B%E7%89%A9%E6%96%99">起始物料</a> 的步骤开始，都适用 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q7A_Good_Manufacturing_Practice_Guide_for_Active_Pharmaceutical_Ingredients">ICH Q7</a> 的 GMP 条款。在 GMP 条件下进行生产，并结合适当的控制策略可为原料药的质量提供保证。；</li></ul><ul><li>起始原料应当是一种具备明确化学特性和结构的物质。未分离的中间体通常不被考虑作为合适的起始原料；</li></ul><ul><li>起始原料作为重要的结构片段并入原料药的结构中。此处所说的“重要结构片段”用于区分起始原料与试剂、溶剂及其他原材料。用来制备盐、酯或其他简单衍生物的常用化学物质应该被视为试剂。</li></ul><p>在选择起始原料时应当考虑上述全部原则，而不是孤立的严格遵循每一个原则（参见案例 4，第 10.4 部分）。</p><h3>5.1.2 半合成原料药的起始原料选择</h3><p>根据本指导原则，半合成原料药是一种通过将生物来源的成分（例如，通过发酵或从植物中提取的方式获得）与化学合成方法相结合，从而引入结构组成的物质。在一些情况下，对申请人而言，从源物质（微生物或植物）开始描述生产工艺可能较为合适。不过，如果能够证明合成工艺中的一个分离中间体符合上述化学合成原料药起始原料的选择原则，则该分离中间体可被提议作为起始原料。申请人应当对所提议的起始原料能否进行分析表征进行特别评估，包括其杂质谱，以及发酵过程、植物和提取过程是否对原料药的杂质谱产生影响。还应该论述来源于微生物和其他污染物的风险。</p><h3>5.1.3 生物技术/生物原料药的源物质和起始原料选择</h3><p>细胞库是制造生物技术原料药和一些生物原料药的起点。在某些地区，其被看作源物质；在其他地区，则被看作起始原料。在 ICH 指导原则 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5A_R2_Viral_Safety_Evaluation_of_Biotechnology_Products_Derived_from_Cell_Lines_of_Human_or_Animal_Origin">Q5A</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5B_Analysis_of_the_Expression_Construct_in_Cells_Used_for_Production_of_r-DNA_Derived_Protein_Products">Q5B</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5D_Derivation_and_Characterisation_of_Cell_Substrates_Used_for_Production_of_Biotechnological/Biological_Products">Q5D</a> 中有相关的指南。</p><h2>5.2 起始原料或源物质的信息申报</h2><p>申请人应当对所提出的所有起始原料或源物质进行确认，并提供合适的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 。应该论证合成和半合成原料药的起始原料的合理性。</p><h3>5.2.1 合成原料药中起始原料的选择依据</h3><p>申请人应当根据 5.1.1 节总则就每一个起始原料的合理性进行论证。可包含以下信息：</p><ul><li><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 检测起始原料中 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> 的能力；</li></ul><ul><li>在后续工艺步骤中，杂质及其衍生物的去向和清除；</li></ul><ul><li>每个起始原料的拟定质量标准将如何有助于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 。</li></ul><p>作为合理性论证的一部分， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 应提供一个现行的原料药生产合成路线流程图，其中应清晰地标出所提议的起始原料。起始原料 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> ，以及从起始原料到最终原料药的合成路线的变更，必须满足当地的、批准后变更的要求。另外，涉及起始原料供应商的地区性要求也可能适用。</p><p>如果使用市售的化学品作为起始原料，申请人通常不需要对其合理化进行论证。市售化学品通常除了用作药物的起始原料外，还已经在非药用市场进行销售。定制合成制备的化学物质不属于市售化学品。如果将定制合成的化学物质用作起始原料，需要依照第 5.1.1 节的总则对其合理性进行论证。</p><p>在一些情况下，原料药生产企业需增加起始原料的精制步骤，以确保市售来源起始原料质量的一致性。在这种情况下，所增加的精制步骤应该作为原料药生产工艺的一部分来进行描述。通常需要提供外购的以及精制后的起始原料的质量标准。</p><h3>5.2.2 半合成原料药起始原料的选择依据</h3><p>如果提议将分离得到的中间体作为半合成原料药的起始原料，申请人应该提供合理性论证，解释所提议的起始原料符合上述 5.1.1 部分中列出的起始原料选择总则。否则，申请人应该从合适的微生物或植物开始描述生产工艺，而且这些物料应当是合格的。</p><h3>5.2.3 生物技术/生物原料药中源物质和起始原料的条件认定</h3><p>在 ICH 指导原则 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5A_R2_Viral_Safety_Evaluation_of_Biotechnology_Products_Derived_from_Cell_Lines_of_Human_or_Animal_Origin">Q5A</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5B_Analysis_of_the_Expression_Construct_in_Cells_Used_for_Production_of_r-DNA_Derived_Protein_Products">Q5B</a> 及 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5D_Derivation_and_Characterisation_of_Cell_Substrates_Used_for_Production_of_Biotechnological/Biological_Products">Q5D</a> 中有相关的指南。</p><h2>6. 控制策略</h2><h2>6.1 总则</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 是源于对当前产品与工艺的理解、确保工艺的进行和产品质量的一系列有计划的控制（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）。每个原料药的生产工艺，无论是通过传统方式开发还是通过增强方式开发（或者两者兼有），都有相关的控制策略。</p><p>控制策略可以包括但不限于以下内容：</p><ul><li>对物料属性的控制（包括原材料、起始原料、中间体、试剂、直接接触原料药的内包材等）；</li></ul><ul><li>内含在生产工艺设计中的控制（例如，精制步骤的顺序[生物技术/生物原料药]，或试剂的加入顺序[化学实体药物]）；</li></ul><ul><li>过程控制（包括过程检测及工艺参数）；</li></ul><ul><li>原料药的控制（例如放行检测）。</li></ul><h3>6.1.1 控制策略的开发方法</h3><p>控制策略的开发可以结合各种方法进行，对一些关键质量属性、步骤或单元操作采用传统的方式；对其他方面采用增强的方式。</p><p>在传统的生产工艺开发及控制策略中，通常为了确保生产的一致性，基于试验数据所设定的参数及操作范围通常都很窄。传统方式更侧重于对原料药阶段的关键质量属性的评估（即最终产品的检测）。在操作范围变化方面（例如，在原材料中），传统方式仅能提供有限的灵活性。</p><p>与传统方式相比，采用增强方式开发的生产工艺可以获得对工艺和产品更深入的理解，因而可用更加系统的方法来确认发生变化的根源。这就允许开发更有意义和更有效的参数、属性和过程控制。在产品的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 中，随着对工艺的理解水平不断加深，可以通过多次循环的方式来开发控制策略。在解决操作范围的变化情况方面（例如，在原材料中），基于增强方式的控制策略可以为工艺参数提供灵活的操作范围。</p><h3>6.1.2 开发控制策略方面的考虑</h3><p>控制策略应该保证每个原料药的关键质量属性处于合适的范围、限度或分布区域内，以确保原料药的质量。原料药的质量标准是整个控制策略的一部分，而且并非所有的关键质量属性都需要被列入原料药的质量标准中。关键质量属性可以是：（1）包括在质量标准中，并通过对最终原料药的检测加以确定；或（2）包括在质量标准中，并通过上游控制（例如，通过实时放行检测[RTRT]）加以确定；或（3）不包括在质量标准中，但通过上游控制来提供保证。上游控制的例子可包括：</p><ul><li>过程检测；</li></ul><ul><li>对能够预测 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> 的工艺参数和/或过程中的物料属性进行测定。在一些情况下，可用过程分析技术（PAT）来加强过程控制，保证产品的质量。</li></ul><p>无论采用传统的还是增强的工艺开发方式，使用上游控制策略应基于对关键质量属性变化的来源的评估和理解。还应该考虑可能对原料药质量产生影响的下游因素，诸如温度的变化、氧化条件、光照、离子浓度含量和剪切力。</p><p>生产企业在进行控制策略开发时，可以根据与关键质量属性相关的风险，以及单一控制检测潜在问题的能力，考虑在工艺中对某个特定的关键质量属性实行单点或多点控制。例如，对于无菌化学实体或生物技术/生物原料药，低水平细菌或病毒污染的检测存在固有的局限性，在这种情况下，仅对原料药进行检测不能充分保证其质量，故需要在控制策略中增加额外的控制（例如，物料控制及过程控制）。</p><p>生产工艺中的每种原材料的质量都应满足其预期使用的要求。接近生产工艺末端使用的原材料比上游使用的原材料更有可能将杂质引入原料药。因此，生产企业应评估是否应该对此类物料的质量采取比上游使用的类似物料更加严格的控制。</p><h2>6.2 控制策略信息的申报</h2><p>所提供的控制策略方面的信息应包括对控制策略单个要素的详细描述，在适当的时候，还要加上原料药控制策略的整体总结。可以采用表格形式或图例形式，这样比较直观化，便于理解（参见第 10.5 部分的案例 5，以列表的形式给出的控制策略总结实例）。理想的总结应能够解释控制策略的各个要素是如何共同作用以确保原料药的质量。</p><p><a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH M4Q</a> 推荐控制策略的每个要素在申报资料的合适章节进行报告，包括：</p><ul><li>生产工艺的描述与工艺控制（3.2.S.2.2）</li></ul><ul><li>物料控制（3.2.S.2.3）</li></ul><ul><li>关键步骤和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E9%97%B4%E4%BA%A7%E5%93%81">中间体</a> 的控制（3.2.S.2.4）</li></ul><ul><li>原料药的控制（3.2.S.4）</li></ul><ul><li>容器密闭系统（3.2.S.6）</li></ul><h2>7. 工艺验证/评价</h2><h2>7.1 总则</h2><p>工艺验证是工艺在设定参数范围内运行时，能有效地、可重复地运行以制备出符合预定质量标准及质量属性的原料药或中间体的书面证据（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q7A_Good_Manufacturing_Practice_Guide_for_Active_Pharmaceutical_Ingredients">ICH Q7</a> ）。</p><p>工艺验证包括从工艺设计阶段开始、贯穿于整个生产过程中的数据收集和评估，以确立所用工艺能持续生产出合格原料药的科学证据。</p><p>原料药的生产工艺验证应在其制剂产品上市销售前进行。对于生物技术工艺，或原料药的无菌工艺及灭菌工艺步骤，上市申报的申报资料中应包含支持工艺验证的数据（3.2.S.2.5）；对于非无菌化学实体原料药的工艺，申报文件一般不包括工艺验证的研究结果。</p><p>一般来说，工艺验证包括适当生产批次的数据收集（参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q7A_Good_Manufacturing_Practice_Guide_for_Active_Pharmaceutical_Ingredients">ICH Q7</a> 的第 12.5 部分）。批次数量取决于但不限于以下几个因素：（1）所验证工艺的复杂性；（2）工艺的可变性程度；（3）在特定工艺条件下获得的实验数据和/或工艺知识的数量。</p><p>作为传统 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E9%AA%8C%E8%AF%81">工艺验证</a> 的替代方法，持续的工艺确认（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> ）可以用于最初的商业化生产的工艺验证方案中，同样也可用于产品生命周期剩余阶段的生产工艺变更中，以进行持续的改进。</p><h2>7.2 生物技术/生物原料药的特定原则</h2><p>对于生物技术/生物原料药，申报文件提供的支持工艺验证的信息通常包括商业化规模的工艺验证研究和小规模的研究。考虑到在工艺描述中所详细描述的批的定义，工艺验证的批次应该能够代表商业化工艺。</p><p>来自小规模研究的数据对全部验证工作的贡献大小将取决于小规模研究的模型能否恰当地代表拟定的商业化规模情况。提供的数据应能说明该模型具有可扩展性，能够代表所拟用的商业工艺。小规模模型适用性的成功应用，可以使生产者减少工艺验证对商业化规模批次的依赖。从商业化规模批次中获得的数据应能对这些小规模研究的结果进行确认。科学的根据、或参照指导原则不需要或明确排除此类研究，可以作为仅开展小规模的特定研究（例如，病毒的去除）的合理依据。</p><p>应开展研究来证明工艺具有去除与产品相关杂质、与工艺相关杂质（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">ICH Q6B</a> ）及潜在污染物（例如在工艺中使用来源于人或动物的物料中的病毒，参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5A_R2_Viral_Safety_Evaluation_of_Biotechnology_Products_Derived_from_Cell_Lines_of_Human_or_Animal_Origin">ICH Q5A</a> ）的能力。证明色谱柱使用周期的研究可包括在小规模模型上进行的实验研究，但应当在商业规模的生产上进行确认。</p><p>应当评估 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%95%86%E4%B8%9A%E5%8C%96%E5%A4%A7%E7%94%9F%E4%BA%A7">商业化生产</a> 中所使用的体外细胞的寿命限度。ICH 文件 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5B_Analysis_of_the_Expression_Construct_in_Cells_Used_for_Production_of_r-DNA_Derived_Protein_Products">Q5B</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5D_Derivation_and_Characterisation_of_Cell_Substrates_Used_for_Production_of_Biotechnological/Biological_Products">Q5D</a> 中对相关产品提供了进一步的指导。</p><p>当应用平台生产经验时，应该对控制策略的适用性进行论证，且在提交上市申请时，原料药的生产工艺应该被适当地验证。通常情况下，完整的规模验证研究应包括来自用于生产商业化产品的最终生产工艺和生产地点的数据资料。</p><h2>8. 采用通用技术文档（CTD）格式撰写的生产工艺开发及相关信息的申报</h2><p>CTD 中并没有规定使用增强方式进行工艺开发所获得的信息在文件中的位置。工艺开发的信息通常在 CTD 的3.2.S.2.6 部分提交。其他从开发研究中获得的信息可以通过多种不同的方式纳入 CTD 格式的文件中，下面给出了几点具体的建议。申请人应明确指出不同信息所在文件中的位置。除了申报资料中提交的信息外，本指导原则所参照的特定主题（例如，生命周期管理、持续改进）应安排在申请人的药物质量体系下（PQS，参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）。</p><h2>8.1 质量风险管理与工艺开发</h2><p>质量风险管理可以被用于从工艺开发到最终生产之间的不同阶段。用于指导和证明开发决策的评估（例如，风险评估以及物料属性和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%8F%82%E6%95%B0">工艺参数</a> 与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> 之间的功能性关系）可以在第 3.2.S.2.6 部分进行概述。</p><h2>8.2 关键质量属性（CQAs）</h2><p>应列出原料药关键质量属性，并在申报资料的生产工艺开发部分（3.2.S.2.6）提供指定这些性质或特性作为关键质量属性的理由。不过，应该在 CTD 格式资料的合适部分提供支持指定这些性质或特性作为关键质量属性的结构特性研究的详细信息（例如，3.2.S.3.1 结构解析及其他特性，3.2.S.7 稳定性）。一些关于原料药关键质量属性的讨论，因其与药物制剂的关键质量属性相关，可以在申报资料的制剂开发的部分提供（3.2.P.2.1 药物制剂中的组分）。</p><h2>8.3 设计空间</h2><p>作为提议的生产工艺的一个要素，设计空间可以在申报资料的生产工艺描述和过程控制（3.2.S.2.2）部分进行描述。适当情况下，可在申报资料的关键步骤和中间体的控制（3.2.S.2.4）部分提供补充信息。申报资料的生产工艺开发部分（3.2.S.2.6）适合对建立设计空间的工艺开发研究进行总结和描述。设计空间与整体控制策略的关系可在申报资料的原料药质量标准的制定依据部分（3.2.S.4.5）进行讨论。</p><h2>8.4 控制策略</h2><p>尽管原料药的质量标准仅是整体控制策略的一部分，但包含原料药质量标准制定依据的部分（3.2.S.4.6）是一个很好的章节用来总结原料药的全部控制策略。然而，关于输入物料控制、工艺控制及原料药控制的详细信息仍应该在 CTD 格式资料的合适部分提供（例如，生产工艺描述和过程控制[3.2.S.2.2]、物料控制[3.2.S.2.3]、关键步骤和中间体的控制[3.2.S.2.4]、原料药的质量标准[3.2.S.4.1]）。关于生产工艺发展过程中的控制策略的相关 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> 方面的简述应该在申报资料的 3.2.S.2.6 部分提供。</p><h2>9. 生命周期管理</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> 描述的质量体系要素和管理职责，意在鼓励在生命周期的每一个阶段使用基于科学和基于风险的方法，以促进整个产品生命周期的持续改进。产品和工艺知识方面的管理应该从开发开始，贯穿产品的整个商业生命周期，直至产品的退市。</p><p>原料药生产工艺的开发与改进通常持续在其整个生命周期。应该对生产工艺的执行情况，包括对控制策略的效果进行定期评估。从商业化生产中获得的知识可以用来进一步加深对工艺的理解和工艺性能，并对控制策略进行调整以确保原料药的质量。从其他产品或革新技术中获得的知识也有助于这些目标的完成。持续的改进和成功的工艺验证，或连续的工艺确认，需要适宜和有效的控制策略。</p><p>应当采用系统的方法来管理生命周期内涉及原料药及其生产工艺的知识。知识管理应该包括但不限于工艺开发活动、内部场地与协议生产商的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8A%80%E6%9C%AF%E8%BD%AC%E7%A7%BB">技术转移</a> 活动、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 的工艺验证研究、以及变更管理活动。应根据需要对知识和工艺理解进行分享，以在原料药生产所涉及的不同产地实施生产工艺和开展控制策略。</p><p>在最初的申报资料中，申请人可以提交一个在产品的生命周期中如何对未来特定的变更进行管理的方案，第 10.2 部分的案例 2 中给出了一个生物技术产品工艺参数的生命周期管理方面的案例。</p><p>对生产工艺提出的任何变更都应评估其对原料药质量及制剂产品（如果适用）的影响。评价应当基于对生产工艺的科学理解，并进行适当的检测以分析该变更的影响。对于化学实体，用于分析变更影响的合适检测应该包括、但不限于评估现有的以及潜在的新杂质，并评估测试方法对任何新杂质的检测能力。在变更实施以后，应在工艺中寻找一个合适点开展该项检测（例如，对中间体或原料药）。关于生物技术/生物学原料药的工艺变更，参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q5E_Comparability_of_Biotechnological/Biological_Products_Subject_to_Changes_in_their_Manufacturing_Process">ICH Q5E</a> 。</p><p>作为质量体系的一部分（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q7A_Good_Manufacturing_Practice_Guide_for_Active_Pharmaceutical_Ingredients">ICH Q7</a> 与 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ），所有的变更都应归于内部变更程序。包括不需要被地区监管部门批准的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E8%AE%A1%E7%A9%BA%E9%97%B4">设计空间</a> 内部的改变。</p><p>改变档案文件中已存档的信息和已批准的信息，应按照地区法规和指导原则的要求向监管部门报告。</p><h2>10. 2：采用质量风险管理来支持工艺参数的生命周期管理</h2><p>本案例阐释了一个重复的质量风险评估是如何被用于沟通分类的基本原则和提议将来工艺参数变更的管理。在这个风险分级柱状图中给出了建立 Q-阴离子交换柱的设计空间的相关参数。柱状图中展示的参数分级目的仅是作为例证的目的，并未包括全部的参数，也不意味着适用于使用离子交换色谱法的所有产品。</p><p>首次申报</p><p>运用之前的知识和开发研究的质量风险评估可以被用于工艺参数的分级，基于如果参数范围发生改变时，他们对产品质量影响的相对可能性。这个柱状图显示了在提交资料时基于知识和理解，将来参数范围的变更对质量的潜在影响。 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%BC%80%E5%8F%91">工艺开发</a> 研究和相互作用研究引导确立了影响 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> 的各个较高风险参数（参数 A-F）的设计空间边界。参数G、H 和 I 也在开发研究的过程中进行了边界挑战，显示在条件研究中对关键质量属性没有影响。这些参数范围的改变仍会带来残留风险（基于之前的知识/不确定性因素，包括潜在的规模敏感性）。参数 J-T 被认为是低风险参数，这基于已有记录的之前的知识，因而预期不会对质量属性产生影响。从质量风险评估得到的参数分级可以被用于与监管者交流沟通，可以看作一种生命周期的管理方法来确保产品生命周期中的持续改进。</p><p>生命周期管理选项</p><p>由于对生产过程的理解不断深入，风险在整个生命周期中应该被重新评估。关于生命周期管理变更的建议可以在 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> 中所述的药品质量体系（PQS）中被找到。</p><p>在设计空间内进行的操作不应被认为是变更。在设计空间外进行的操作被视为变更，因此对于较高风险参数（如参数 A-F），在设计空间之外的范围延伸将通常需要启动批准后变更的申报流程。</p><p>申请人在最早提交的资料中应该包括一个明确的对于参数 G、H 和 I 将来在产品的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 发生变更后的管理建议。对于低风险参数（J-T）的范围延伸主要通过 PQS 来处理，不需要事先的监管方批准，尽管根据地区监管和指导原则的要求可能需要进行通告。如果随后决定在文件归档后对风险分级方面进行变更，如此参数范围的延伸代表了更高的风险，这个变更应通过地区监管程序进行合适的申报。</p><div><a href="https://lib.shilinx.com/wiki/images/9/97/CDE_ICH_Q11_%E5%8E%9F%E6%96%99%E8%8D%AF%E5%BC%80%E5%8F%91%E5%92%8C%E7%94%9F%E4%BA%A7_%E5%8C%96%E5%AD%A6%E5%AE%9E%E4%BD%93%E5%92%8C%E7%94%9F%E7%89%A9%E6%8A%80%E6%9C%AF-%E7%94%9F%E7%89%A9%E5%AE%9E%E4%BD%93%E8%8D%AF%E7%89%A9_20120501-6.jpg"><img src="https://lib.shilinx.com/wiki/images/9/97/CDE_ICH_Q11_%E5%8E%9F%E6%96%99%E8%8D%AF%E5%BC%80%E5%8F%91%E5%92%8C%E7%94%9F%E4%BA%A7_%E5%8C%96%E5%AD%A6%E5%AE%9E%E4%BD%93%E5%92%8C%E7%94%9F%E7%89%A9%E6%8A%80%E6%9C%AF-%E7%94%9F%E7%89%A9%E5%AE%9E%E4%BD%93%E8%8D%AF%E7%89%A9_20120501-6.jpg" width="400" height="156"></a></div><p>本案例说明在选择合适的起始物料时需要考虑 5.1.1 部分所描述的所有基本原则的重要性，而不是孤立的应用其中某一个原则。本案例是虚构的，基于采用线性合成策略合成一个相对简单的分子，该案例无意传达任何关于反应步骤数的特殊含义。</p><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 中所需的立体化学构型在第一步反应化合物B 的合成中产生，反应中使用了一个外购的非手性前体 A 和一个手性选择试剂。在第一步反应中还会生成少量化合物 B 的对映异构体。一旦生成，两个立体化学构型都会遵循相同的后续合成步骤，因而原料药中也含有少量不需要的异构体作为特定杂质。通常影响原料药杂质谱的生产步骤应被包含在申报资料的 3.2.S.2.2 部分中所述的生产过程中，按照这一原则，可能得到的结论是步骤 1 应在 3.2.S.2.2 部分中进行工艺描述，而化合 A 应被考虑作为起始原料。</p><p>然而，对于这个生产过程，我们还知道原料药中的所有重要杂质（除对映异构体外）皆在步骤 4、5 和 6 中产生，步骤 2 和步骤 3 对原料药的杂质谱无影响，步骤 1 对原料药的唯一影响就是对映异构体杂质。此外，我们还知道在第一步中最早形成的手性中心在所有后续步骤的生产条件下稳定（例如：未发生过消旋化或不太可能发生消旋化），而且还有合适的分析方法来测定化合物 D 中对映异构体的量。因此，如果化合物 D 符合大部分 5.1.1 部分中所述的一般原则，那么，根据生产过程的前期步骤往往较后期步骤对原料药的质量影响更小这一原则，提议化合物 D 取代化合物 A 作为起始原料应是合理的。在这个案例中，步骤 1 对原料药的唯一影响是其中的异构体杂质的量，这可以通过在化合物 D 中设定一个对映异构体的合适限度的方法来控制。关于步骤 1-3 的信息应按照不同国家的要求递送至监管当局来论证该提议的合理性。</p><p>如果原料药中的手性中心源于外购前体 A，而不是在步骤 1 中产生，也要进行类似的论证。</p><h2>10.5 案例 术语</h2><p><b>化学转化步骤（Chemical Transformation Step）</b> ：对于化学实体药物，指从其前体分子片段合成原料药化学结构的反应步骤。典型的有 C-X 或 C-C 键的形成或断裂。</p><p><b>污染物（Contaminants）</b> ：引入的任何外源性物质（如化学、生化物质或微生物），而不是原料药或制剂生产过程中的一部分。（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">ICH Q6B</a> ）</p><p><b>持续工艺确认（Continuous Process Verification）</b> ：工艺验证的另一种方法，持续监控和评价生产工艺性能。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> （Control Strategy）</b> ：为确保工艺性能和产品质量而计划进行的一系列控制，来自于当前对产品和工艺的了解。这些控制可包括与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 以及药物制剂的材料和组分相关的参数和属性， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E6%96%BD">设施</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E5%A4%87">设备</a> 运行条件，过程控制，成品质量规范，以及相关的监测和控制方法与频率。（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> （Critical Quality Attribute，CQA）</b> ：指产品的物理、化学、生物或微生物性质或特性，应在适当的限度、范围或分布之内，以确保预期的产品质量。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%AE%BE%E8%AE%A1%E7%A9%BA%E9%97%B4">设计空间</a> （Design Space）</b> ：设计空间是已被证明有质量保障作用的物料变量（如物料属性）和工艺参数的多维组合和交互作用。在设计空间内的操作，不视为变更。而超出设计空间的运作，则应视为变更，并应进行上市后的变更申请。设计空间由申报者提出，送交管理当局审评并批准（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> ）。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E9%97%B4%E4%BA%A7%E5%93%81">中间体</a> （Intermediate）</b> ：参见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q7A_Good_Manufacturing_Practice_Guide_for_Active_Pharmaceutical_Ingredients">ICH Q7</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%9D%82%E8%B4%A8">杂质</a> （Impurity）</b> ：参见 <a Q6A</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">ICH Q6B</a> 。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> （Lifecycle）</b> ：一个产品从开始开发到上市，直至产品终止的所有阶段（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> ）。</p><p><b>平台制造（Platform Manufacturing）</b> ：新药的生产发展策略方式，起始时所用的生产过程与同一申报人在生产相同类型的其他药物时相似（例如：在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8D%95%E5%85%8B%E9%9A%86%E6%8A%97%E4%BD%93">单克隆抗体</a> 的生产中，使用预先确定的宿主细胞、细胞培养和纯化过程，这些已经有了相当多的经验）</p><p><b>工艺的稳健性（Process Robustness）</b> ：工艺承受物料的波动及工艺与设备改变，而不造成负面质量影响的能力。</p><p><b>质量风险管理（Quality Risk Management，QRM）</b> ：贯穿产品生命周期的药品质量风险的评估、控制、通报和回顾的系统化过程。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:QTPP">目标产品质量概况</a> （Quality Target Product Profile，QTPP）</b> ：指理论上可以达到的，关于药品质量特性的前瞻性概述，用以保证所需的质量要求，需考虑药品的安全性和有效性。</p><p><b>实时放行检验（Real Time Release Testing，RTRT）</b> ：指根据工艺数据评价并确保中间产品和/或成品质量的能力，通常包括已测得物料属性和工艺控制的有效结合。</p><p><span>评论</span></p></td></tr></tbody></table></div></td></tr></tbody><tfoot><tr><td></td></tr></tfoot></table>
