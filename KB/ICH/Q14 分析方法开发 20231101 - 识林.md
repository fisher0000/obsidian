目的</h2><p>本指导原则描述了基于科学和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%A3%8E%E9%99%A9%E7%AE%A1%E7%90%86">风险管理</a> 进行 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95"><i>分析方法</i></a> 开发和维护的方式，以用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 和制剂的质量评价。 <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8“药品研发”</a></i> 中建议的系统方法与 <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q9_R1_Quality_Risk_Management_20230118">ICH Q9“质量风险管理”</a></i> 中的原则也可用于分析方法的开发和生命周期管理。进行分析方法开发时，可使用基础（即传统）方式或增强方式的要素。 此外，指导原则还阐述了 <i>多变量分析方法开发</i> 和 <i>实时放行检测（RTRT）</i> 的其他注意事项。</p><p>本指导原则是对 <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2“分析方法验证”</a></i> 的补充。</p><p>使用 <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12“药品生命周期管理的技术和监管考虑”</a></i> 中所述工具，基于风险管理、对分析方法的全面理解以及符合 <i>性能特征</i> 的预设标准，本指导原则描述了分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> 管理原则。应用增强方式进行分析方法开发所获得的知识，可更好地保证方法性能，可作为 <i>分析方法控制策略</i> 的基础，并且有机会使相关批准后变更的监管方法更加高效。</p><p>本指导原则还说明了在通用技术文档（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">CTD</a> ）格式（ <a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality"><i>ICH M4Q“人用药物注册通用技术文档</i></a> ）中分析方法开发和相关 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 信息的提交。可向监管机构提交与分析方法开发相关的信息和知识，从而提供分析方法适用于预期目的的进一步证据。尽管基础方式依旧是一种认可的方式，但 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 也可以决定提交额外的开发数据和知识，其可能有助于与监管机构就批准后变更管理进行沟通。</p><h2>1.2 范围</h2><p>本指导原则适用于商业化的原料药和制剂（下文简称为“产品”）放行和稳定性检测所用的分析方法。基于风险评估，本指导原则也适用于作为 <i>控制策略（</i> <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10“药品质量体系 <i>”</i></a> ）部分使用的其他分析方法。本指导原则中描述的科学原则可采用与阶段相适应的方式用于临床开发期间所用的分析方法。</p><h2>2. 分析方法的一般考虑</h2><p>开发目的是获得适用于预期目的的分析方法：在可报告范围内，按照所需的 <i>专属性/选择性、准确度、精密度</i> ，测量被分析物一项或多项属性。分析方法验证中考虑的性能特征描述详见 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> 。</p><p>本节描述了分析方法开发的基础方式和增强方式。尽管基础方式依旧是开发适用于预期目的的耐用分析方法的可接受方式，但增强方式的部分或全部要素可用于支持分析方法的开发和生命周期管理。</p><p>在某些情况下，测定条件略作修订或不修订，分析方法就可应用于多种产品。对于此类 <i>平台分析方法</i> 的新应用，后续开发可进行简化，并且根据科学和风险评估的合理依据，可省略某些 <i>验证试验</i> 。</p><p>开发研究期间获得的数据（例如，来自实验设计（DoE）研究的 <i>耐用性</i> 数据）可用作相关分析方法性能特征的验证数据的一部分，不一定需要重复研究。</p><h2>2.1 分析方法开发的基础方式与增强方式</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 开发应至少包括以下要素：</p><ul><li>确定需要检测的产品属性；</li></ul><ul><li>选择适当的技术及相关仪器或适宜设备；</li></ul><ul><li>开展研究以对分析方法的性能特征进行评价，例如在可报告范围内的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%93%E5%B1%9E%E6%80%A7">专属性</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> （包括 <i>校正模型</i> 、范围上限和/或下限）和耐用性；</li></ul><ul><li>记录分析方法，包括分析方法的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 。</li></ul><p>增强方式提供了一种系统的方式来开发分析方法、完善分析方法的知识，并证明对分析方法的理解。对产品和工艺的理解可以确定待检测的质量属性。相关性能特征的预期 <i>性能标准</i> 应记录于 <i>分析目标概况</i> （ATP）中。除了基础方式中所述要素之外，增强方式还包括下列一项或多项要素：</p><ul><li>进行风险评估和评估先验知识，以识别可能影响方法性能的 <i>分析方法参数</i> 。</li></ul><ul><li>进行单变量或多变量实验和/或建模，以探索已确定的分析方法参数的范围和相互作用。</li></ul><ul><li>拟定分析方法控制策略，包括相关分析方法参数的设定值和/或范围。这些可能包括 <i>已证实的分析方法可接受范围</i> （ PAR ）和/ 或 <i>方法可操作设计区域</i> （MODR）。</li></ul><p>将增强方式的要素应用于分析方法开发，可更好地理解分析方法参数对分析方法性能的影响，并为 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 管理提供更大的灵活性，例如更宽的操作范围、一组更为合适的 <i>既定条件（ECs）</i> 及相关报告类别。</p><h2>2.2 分析方法生命周期</h2><p>图1描述了与产品生命周期相关的分析方法生命周期的要素。本指导原则描述了分析方法开发和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4%E7%AE%A1%E7%90%86">变更管理</a> 方式，而 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法验证</a> 在 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> 中描述。根据分析方法的预期目的和所采用的开发方式，各要素的顺序和程度可能有所不同，并且多个要素可能同时存在。</p><h2>3. 分析目标概况</h2><p>根据对产品和工艺的理解（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q11_Development_and_Manufacture_of_Drug_Substance">ICH Q11“原料药开发和生产”</a></i> ），可识别出需要分析测量控制的 <i>关键质量属性</i> （ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">CQA</a> ），这些属性可能包含在目标产品质量概况（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:QTPP">QTPP</a> ）中。测量需求可记录到ATP中，这构成了分析方法开发的基础。ATP包括对分析方法预期目的的描述、待测产品属性的适当细节、以及性能特征及相关性能标准。ATP包括单个或多个质量属性的测量要求。ATP决定分析技术的选择。多种可用的分析技术可能符合性能标准。在技术选择时应考虑操作环境（例如，近线、在线或离线）。一旦选定技术之后，ATP可用于获得适当 <i>分析方法属性</i> 和分析方法验证性能标准的基础（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）。ATP的正式记录和提交并非强制性要求，但无论选择何种开发方法，ATP的正式记录和提交都有助于促进监管沟通。</p><p>ATP还有助于分析方法的 <i>持续监测</i> 和持续改进。在整个生命周期内对ATP进行维护，ATP也可用作生命周期管理的依据，以确保现有的、修订的或新的分析方法始终适用于预期用途。</p><p>ATP的示例见附录A。</p><h2>4. 分析方法开发和持续改进过程中的知识管理和风险管理</h2><h2>4.1 知识管理</h2><p>与产品和生产工艺开发相同， <i>知识管理</i> （ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）在分析方法开发和分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 管理中起到关键作用。</p><p>在分析方法开发和生命周期管理中，先验知识可直接或间接地用于指导决策。先验知识可以是来自公司自有开发和分析经验的内部知识、外部知识如参考科学和技术出版物或已确定的科学原理。</p><p>先验产品知识在确定适当的分析技术方面具有重要作用。对最佳做法、最先进技术以及监管预期的了解，有助于为特定目标选择最合适的技术。可利用现有平台分析方法（例如，通过紫外光谱法测定蛋白质含量）对特定产品的属性进行评价，而无需进行额外的方法开发。</p><p>随着不断获取新的信息，应在整个产品生命周期内积极管理分析方法相关知识。</p><h2>4.2 风险管理</h2><p>鼓励通过 <i>质量风险管理</i> （ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%A3%8E%E9%99%A9%E7%AE%A1%E7%90%86">QRM</a> ）帮助开发耐用的分析方法，从而降低性能不佳和报告结果不正确的风险。风险评估通常在分析方法开发早期阶段进行，在得到更多新信息时应进行更新。风险评估可以是正式的，也可以是非正式的，并可使用先验知识支持。</p><p><i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q9_R1_Quality_Risk_Management_20230118#Annex_I:_Quality_Risk_Management_Methods_and_Tools">ICH Q9附录1</a></i> 中所述风险评估工具可用于</p><ul><li>确定可能影响分析方法性能的分析方法参数（因子和操作步骤），例如附录A 图2（Ishikawa图）；</li></ul><ul><li>评估分析方法参数对分析方法性能的潜在影响；</li></ul><ul><li>确定需进行实验研究的分析方法参数并考虑优先级；</li></ul><ul><li>确定 <i>持续监测</i> 的需求和范围，作为风险审查的一部分；</li></ul><p>风险沟通应用于支持分析方法性能在整个生命周期内的持续改进。质量风险管理的结果应记录到 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 的药品质量体系（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E4%BD%93%E7%B3%BB">PQS</a> ）的相关部分（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a> ）。</p><h2>5. 分析方法的耐用性和参数范围评估</h2><h2>5.1 耐用性</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的耐用性用于衡量正常使用期间满足预期性能标准的能力。耐用性是通过有意改变分析方法参数来测试，应考虑分析持续时间（包括样品制备及试剂稳定性）。在耐用性研究期间，先验知识和风险评估可为研究参数的选择提供参考。应研究那些在预期使用期间可能影响分析方法性能的参数。</p><p>对于大多数分析方法，耐用性评价是在开发期间进行的。如果开发期间已经进行了耐用性评价，则根据 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ，在验证期间无需重复进行评价。验证研究数据（例如， <i>中间精密度</i> ）可作为耐用性评价的补充。对于一些固有参数变异性较高的分析方法（例如，需要生物试剂的分析方法），在耐用性研究期间可能需要研究更宽的参数范围。多变量方法的耐用性可能需要更多考虑（参见第8章）。应记录耐用性评价结果，并在分析方法的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 中体现。</p><h2>5.2 分析方法参数范围</h2><p>研究参数范围的试验可提供关于分析方法性能的额外知识。相应的分析方法属性和相关标准可以由ATP确定。单个参数的单变量检测可为分析方法确立PAR。</p><p>在增强方式中，可通过多变量实验（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:DOE">DoE</a> ）对相关参数范围及其相互作用进行研究。应使用风险评估和先验知识确定需要实验研究的分析方法参数、属性以及相关范围。分类变量（例如，不同仪器）也可视为实验设计的一部分。</p><p>开发研究的结果应提供对分析方法参数（输入）和分析方法 <i>响应</i> （输出）之间关系的理解。基于这些结果，可为某些参数规定固定值。对于其他参数，可定义PARs， 剩余的参数则可纳入一个MODR中。一个MODR由两个或多个变量的组合范围组成， 在该范围内分析方法适用于预期目的。</p><p>申请人根据开发和验证数据拟定分析方法参数设定的固定值、PARs 和/ 或MODRs，应取得监管机构批准。在批准的PAR或MODR内变化不需要通知监管机构。</p><p>进行PAR和/或MODR的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法验证</a> 只需要针对分析方法开发数据未能涵盖的性能特征。出于实际原因并遵循基于风险的考虑，可能没有必要或不可能对整个MODR</p><p>进行验证。验证数据必须涵盖分析方法中预期常规使用的PAR或MODR部分（通常为预期操作条件或设定值）。验证试验的范围应根据具体情况进行论证。</p><p>将来对于PAR或MODR范围内操作条件的变更，应对额外验证试验的必要性和程度进行评估。 <i>分析方法验证策略</i> ，如，作为分析方法验证方案的一部分，可以定义进行额外验证的必要程度。</p><h2>6. 分析方法控制策略</h2><p>分析方法控制策略应确保分析方法在其整个 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内的常规使用中适用于预期目的。控制策略由一组控制项组成，这些控制项源自当前对分析方法的理解，包括开发数据、风险评估、耐用性以及先验知识。分析方法控制策略应在验证前设定（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ），并应在验证完成后进行确认。</p><p>分析方法控制策略包括需要控制的分析方法参数和作为分析方法一部分的 <i>系统适用性试验</i> （SST）。分析方法中应描述进行各项分析检测所需的步骤。分析方法描述可能包括（但不限于）样品、 <i>参比物质</i> 和试剂、供试品和对照溶液的制备、仪器的使用、校准曲线的生成、重复次数、计算可报告结果所用公式以及其他必要步骤。描述应达到有经验的分析人员进行分析和结果判断所必需的详细程度（例如，各国药典收载的相似产品标准的详细程度）。</p><p>SST取决于分析方法的类型和目的，通常采用一种或多种预先确定的物质（包括使用阳性对照和/或阴性对照）进行。SST的设计是为了确认所选分析方法的属性。可接受标准应基于分析方法性能标准。应基于风险评估、由开发数据获得的知识和理解来选择SST的组成。在分析期间，系统适用性试验用于确认测量系统、与分析方法相关的分析操作是否适合预期目的，并能够检测不可接受的性能。分析方法结果的有效性取决于SST结果。在增强方式中，为保证方法性能而精心设计一套SST参数及其标准， 可作为降低风险的一个重要措施。对于依赖多变量模型的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> ，应使用合适的软件工具确认数据质量。</p><p>除SST外，可能还需要进行 <i>样品适用性评估</i> ，以确保样品响应可接受。分析方法属性的预设可接受标准是为验证的分析方法（通常用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E7%89%A9%E5%88%B6%E5%93%81">生物制品</a> ）而开发的，如果测定的样品响应值符合分析方法属性的预设可接受标准，则认为样品和/或样品制备液是适用的。在这些情况下，样品适用性与SST结果合格一起作为结果有效的先决条件。样品适用性评估通常包括对参比物质和供试品之间响应相似性的评估，也可能包括样品基质产生干扰信号的可接受水平的要求。对于依赖多变量模型的分析方法，可使用适当的软件工具确认样品适用性评估，这类工具能检查样品是否在模型适用范围内。这一过程通常称为数据质量检查。</p><p>为符合 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E4%BD%93%E7%B3%BB">PQS</a> 预期，建议对所选分析方法的输出结果进行持续监测，以识别任何可能的趋势。对分析方法输出结果进行审核，有助于方法生命周期的管理，且能够主动采取干预措施以避免失效情况。</p><h2>6.1 分析方法的既定条件</h2><p>根据 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a> ，申请人对分析方法可以提出既定条件（ECs）。ECs可使用第2章中提到的工具来识别，工具包括风险评估、先验知识以及单变量和/或多变量实验的结果。ECs的性质和程度取决于分析方法的开发方式、复杂性以及已证明的对于参数和其他因素如何影响分析方法性能的理解。</p><p>采用基础方式进行方法开发时，ECs为固定的分析方法参数和设定值，因此数量可能很多。</p><p>使用增强方式进行方法开发时，应对测量要求、可用技术的适用性以及分析方法参数与性能之间的关系有更多的理解。这些知识有助于识别一组适当的ECs和相关报告类别（见第7章）。当对 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的理解（包括先验知识和产品/工艺知识）和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%A3%8E%E9%99%A9%E7%AE%A1%E7%90%86">风险管理</a> 被证明合理时，可以减少ECs并重点关注分析方法性能（例如，分析方法参数的可接受范围、性能特征和相关标准）。</p><p>ECs可能包括：</p><ul><li>性能特征和相关标准（例如，包含在ATP 中）；</li></ul><ul><li>分析方法原理（即，理化基础或特定技术）；</li></ul><ul><li>SST 和样品适用性评估标准；</li></ul><ul><li>一个或多个分析方法参数的设定值和/或范围。</li></ul><p>需要控制以确保方法性能的分析方法参数以及无法合理排除控制需求的分析方法参数应确定为ECs。如果通过分析方法的性能标准和/或SST的应用证明特定参数处于受控状态，则不一定需要将该参数或参数值定义为EC，或可酌情指定较低的报告类别。ECs和相关报告类别由 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 提出，并由监管机构根据提供的科学依据进行评估，以获得批准。</p><p>采用增强方式时，在注册申报资料中不应导致提供更不详细的分析方法描述。无论采用何种方式确定分析方法的ECs，均应在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">CTD</a> 模块3中以适当的详细程度对分析方法进行描述，以便提供清晰的理解。对分析方法的描述可包括支持性信息以及确定的ECs。</p><p>ECs报告类别的确定以及ECs在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4%E7%AE%A1%E7%90%86">变更管理</a> 中的应用见下一章。</p><h2>7. 分析方法的生命周期管理和批准后变更</h2><p>分析方法在整个产品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内均有可能发生变更，并且变更可能涉及现有方法的修改或包括引入新技术在内的完全替代。性能特征或质量属性附加信息的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%87%8D%E5%A4%A7%E5%8F%98%E6%9B%B4">重大变更</a> 可能导致对ATP的重新评价和/或建立新的方法。通常工艺知识、分析方法知识和持续改进都是变更的推动因素。如果可能，所做变更应使得改进的分析方法达到最佳实践和仪器设置要求。 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a> 中讨论的工具和推动因素适用于分析方法，无论采取何种开发方式，这些工具和推动因素包含：</p><ul><li>现有的基于风险的分析方法变更分类（在区域监管框架内）；</li></ul><ul><li>ECs；</li></ul><ul><li>批准后变更管理方案（PACMP）；</li></ul><ul><li>产品生命周期变更管理（PLCM）文件；</li></ul><ul><li><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E4%BD%93%E7%B3%BB">PQS</a> （记录所有变更，包括不需要向监管机构提交的变更）；</li></ul><ul><li>批准后频繁化学、生产和控制（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CMC">CMC</a> ）变更的结构化方法。</li></ul><p>如果申报资料中未提出ECs，则任何变更都应根据区域性报告要求报告。与基础方式相比，采用增强方式的不同要素有助于批准后变更的管理和与监管机构沟通。</p><p>1）包括分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a></p><p>2）在某些情况下，根据监管机构的反馈，公司提出的中风险 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> 可能需要得到事先批准</p><p>如果已经合理性论证及验证（见第5.2章），PAR或MODR允许在批准范围内操作，并在公司的PQS范围内进行管理。超出批准范围的变更或扩大这些范围需要与监管机构沟通。</p><p>在提出ECs的情况下，应预先评估与预期变更相关的风险，以提出适当的报告类别。需要考虑的因素包括所测量的质量属性的关键性、技术的复杂性以及变更的程度（见附录A）。应根据产品和工艺知识，以及对分析方法的理解和提出的分析方法控制策略，确定相关风险降低措施。最后确定风险水平（高、中或低）。</p><p>一般而言，对 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 耐用性的理解和/或先验知识可用于支持降低与未来变更相关的风险。在提出ECs时，向监管机构提交风险评估结果有助于证明未来分析方法变更报告类别的合理性。</p><p>图2中总结了风险评估和风险降低办法如何有助于确定ECs的适当报告类别。通过定义已确定为ECs的相关性能标准，可以降低与变更相关的风险。当已有足够的理解可用于设计未来的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%A1%A5%E6%8E%A5%E7%A0%94%E7%A9%B6">桥接研究</a> href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4">变更</a> 的影响，以及用于重新确认最初商定的报告类别仍然适当。该风险评估的结果可为支持变更所需研究（包括桥接策略，以证明经修订或新订方法适用于预期目的）的设计和程度提供参考。在考虑桥接策略时，对分析方法有更深入的理解可以帮助研究设计简化，而高风险变更可能需要更加深入的研究（表1）。在不同地点实施已验证的分析方法（包括 <a 多变量分析方法的开发：其他注意事项</h2><p>多变量分析方法是通过多变量校正模型使用多个输入变量来确定结果的方法。本章涉及的考虑因素适用于使用 <i>潜变量</i> （在数学上与直接测定的变量相关）的模型。机器学习（例如，神经网络）或优化技术等其他方法可应用类似原理，尽管具体的方法可能有所不同，此处不做详细讨论。</p><p>然而本章包括多变量分析方法特有的方面，本指导原则其他部分描述的原则也适用。耐用的多变量分析方法的开发过程包括经科学论证的样本选择和范围内分布、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%A0%B7%E6%9C%AC%E9%87%8F">样本量</a> 、模型变量选择和数据预处理。</p><p><b>样本和样本总体</b></p><p>多变量模型将测定的模型变量与从验证过的 <i>参比分析方法</i> 或 <i>参比样品</i> 中获得的数值关联。因此，多变量分析的样本包含输入测定值及其相应参比值，即定量测定（例如，含量测定）的数值和定性分析方法（例如，鉴别）的分类类别。在某些情况下，如果存在多个参比值，则一组输入测定值可用于多个模型。参比值可通过参比分析方法或数值已知的参比样品来测定。应注意，与多变量分析方法的预期性能相比，参比分析方法的不确定性应足够低，并且所制备的参比样品应均匀。应对所用参比方法或所制备的参比样品进行说明，并证明合理性。</p><p>多变量模型的范围通常基于样本数据。因此，谨慎的样本选择策略对于从分析数据中得到相关信息至关重要，并且有助于所建模型的耐用性。根据分析方法和检测原理，样本总体应涵盖生产和分析过程中可能发生的变异度来源，例如原材料质量、生产工艺变异度、储存条件、样品制备和检测。使用风险评估工具有助于识别可能影响测定和最终模型输出结果的变异度来源。</p><p>在商业规模下想要得到具有适当变异度的样本可能不容易。因此，通常使用实验室研发和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E8%AF%95">中试</a> 规模样品提供足够的变异度，以提高模型的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 和耐用性。建议纳入商业规模样本，以获取与特定设备和/或工艺条件相关的变异度。还应仔细考虑 <i>校正</i> 和 <i>验证集</i> 中样本分布，因为这将会影响模型的预测能力。</p><p>创建定量分析校正模型所需的样本数量取决于样本基质的复杂性和/或基质对目标分析物信号的干扰（即，更复杂的样本基质通常需要更多样本）。</p><p>应提供足够的样本来创建具有适当样本量和变异度的校正和验证集。验证集中的 <i>独立样本</i> 不纳入校正集或 <i>内部测试集</i> 。由来自独立批次的样品生成的验证样本集可用于证明模型的耐用性。</p><p><b>数据转换</b></p><p>可根据数据类型、仪器或样品、模型预期目的和/或先验知识选择 <i>数据转换</i> 方法。 在进行任何数据转换时均应谨慎行事，因为可能引入人为影响或丢失必要信息。任何数据转换均应进行记录并说明合理性。</p><p><b>变量选择</b></p><p>在模型开发期间进行变量选择。例如，在光谱应用中常常进行波长范围的选择， 以选择出最能估算待测（待建模）的化学或物理特性的光谱区域。变量的选择取决于测定原理、应用等因素，并应证明其合理性。</p><p><b>耐用性</b></p><p>模型开发应尽量减少预测误差，提供能始终确保长期性能的耐用模型。应通过纳入与材料、工艺、环境、仪器或其他因素相关的变异度来源，将耐用性构建到模型中。可通过先验知识和风险评估来确定变异度来源，并使用统计工具进行评价。耐用性取决于多个因素，例如，校正集的组成、数据转换方法、变量选择和潜变量的数量。</p><p>多变量模型的优化通常需要在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 和耐用性之间进行权衡。在校正模型中使用的潜变量数量是一个关键因素，它可确保模型能够针对预期目的得到优化。在模型开发期间进行潜变量数量的选择，并在 <i>内部测试期间</i> 确认。潜变量过多可能会导致模型过度拟合，可能导致耐用性下降，需要更频繁地更新模型。应提供最终所用潜变量数量的合理性依据。软件包提供的诊断图有助于证明合理性。</p><p><b>重新校正和模型维护</b></p><p>跟踪校正模型的性能是持续监测多变量 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的重要环节。可借助各种统计工具进行诊断，确认模型假设得到支持。对于潜变量模型，这些诊断工具包括：</p><ul><li>残差检验，用于确定数据的未建模特征（例如，x-残差或F-概率）</li></ul><ul><li><i><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%A6%BB%E7%BE%A4%E5%80%BC">离群值</a> 诊断</i> ，用于确定数据是否在模型构建的范围内（例如，霍特林T <sup>2</sup> 或马氏距离）。</li></ul><p>软件包应允许应用诊断工具进行模型预测。</p><p>此外，应定期或在事件驱动下，通过对比模型预测值与参比样品已知值或参比分析方法的结果，确认校正模型的持续性能。该确认性测试有助于确保校正模型持续按预期运行。可能触发确认性测试的事件包括：新发现的工艺变异度、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%9D%9E%E9%A2%84%E6%9C%9F">非预期</a> 的工艺事件或仪器维护计划。</p><p>作为持续改进的一部分，可通过对模型监测触发模型重建（重新校正）。一般而言，需要考虑的因素与原始模型开发和内部测试时相同。基于模型更新的原因（例如， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%8F%98%E6%9B%B4">工艺变更</a> ），可能需要加入新的数据并删除旧的不相关数据。</p><p>新的校正模型建立后，可按照与原始模型中相同的性能标准对更新后的分析方法进行验证。模型更新后预期不会发生变化的方面可能无需进行评价（例如专属性）。</p><p>多变量模型生命周期（图3）是迭代的，可分为3个主要组成部分：模型建立、常规使用和 <i>模型维护</i> 。</p><p>多变量模型的选择应基于分析方法要求和所选的测定技术。模型开发之前，需确定模型的性能因素，包括基础模型假设和期望的模型适用范围。初始风险评估对于了解物料和工艺中可能影响模型性能的潜在变异来源很有价值，因此应在模型校正过程中予以考虑。模型开发（包括校正和内部测试）应遵循本章中概述的注意事项。在模型开发完成后，应使用之前未在校正集中使用过的独立数据进行验证。模型建立的最后一步是开发多变量模型维护计划，其中应包括离群值诊断的方法和限度，并确定确认测试的频率和情形（如需要）。</p><p>多变量分析方法的常规分析通常应包括使用离群值诊断对每次测定的适当性进行监测。如果确认测试或离群值诊断未能满足预定义标准，或数据趋势表明模型、工艺或被测物料存在潜在不可接受的性能，则可能触发模型重新评估（多变量模型 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 组成部分的示例见附录B）。</p><p>在 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E4%BD%93%E7%B3%BB">PQS</a> 内利用知识管理和风险评估进行模型重新评估。如果发现不可接受的性能，可能需要进行模型开发和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> ，例如，在校正集中增加样品，并删除不再相关的样本。在某些情况下，模型可能运行正常，但基于其他经验，可能有必要修改模型维护计划。在其他情况下，发现的不可接受的性能可能与检测系统有关（例如，样品接口未对准），那么无需更新模型。图3中的虚线箭头说明了根据模型重新评估的可能输出重新进入生命周期流程的情形。</p><h2>9. 实时放行检测分析方法的开发：其他考虑</h2><p>实时放行检测（RTRT）是指基于工艺数据评价和确保 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E9%97%B4%E4%BA%A7%E5%93%81">中间产品</a> 和/或成品质量的能力，通常包括已测得物料属性和工艺控制的有效组合（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a> ）。RTRT检测与产品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 的所有要素（例如，工艺监测或过程控制）相结合，以确保产品质量。RTRT 可用于 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E4%B8%AD%E9%97%B4%E4%BA%A7%E5%93%81">中间体</a> 和制剂。</p><p>RTRT可基于一种或多种工艺测定和/或物料属性的适当组合，为一个或多个 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">CQAs</a> 提供数值，并且对这些CQAs具有专属性。应充分论证RTRT方法与产品CQAs以及可接受标准之间的关系。应根据 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> 对RTRT分析方法进行验证，并证明工艺测定对目标质量属性具有适当的专属性。</p><p>在设计任何在线非原位或在线原位分析方法（包括用于RTRT的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> ）时，采样和样品接口都是重要的考虑因素。选择的检测点应能够代表生产中的全部物料，并选择适当的采样持续时间或采样量（例如，相对于单位剂量）。此外，样品接口应在整个生产过程中保持一致，对预期的工艺和环境变化具有稳定性。</p><p>RTRT方法应体现在产品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 中（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8/9/10_Q%26AsR4"><i>ICH 批准的ICH Q8/Q9/Q10 实施指导原则</i></a> ）。根据 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6A_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_New_Drug_Substances_and_New_Drug_Products:_Chemical_Substances_199910">ICH Q6A</a> <i>质量标准：新原料药和新药制剂的检测方法和可接受标准：化学药物</i> 和 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q6B_Specifications_:_Test_Procedures_and_Acceptance_Criteria_for_Biotechnological/Biological_Products">ICH Q6B</a> <i>质量标准：生物技术/生物制品的检测方法和可接受标准</i> ，这其中包括分析方法和相关可接受标准的可用参考。定量RTRT结果应与传统检测采用相同的单位表示。产品质量标准中通常还包括用于离线检测的分析方法。如果申报资料里包括已注册替代RTRT控制策略（例如，当无法进行过程分析时进行的传统终产品检测），则相关分析方法以及何时应用这些方法也应包含在产品质量标准中。</p><h2>10. 分析方法相关信息的提交</h2><h2>10.1 一般监管考虑和文件</h2><p>对 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的描述应包括在 <a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH M4Q</a> <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">CTD</a> 第3.2.S.4.2节（原料药）或第3.2.P.5.2节（制剂）中。证明分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 合理性所需的验证数据和支持性信息应包括在CTD第3.2.S.4.3节（原料药）或第3.2.P.5.3节（制剂）中。控制策略所用其他分析方法应包括在相关CTD章节中（例如，第3.2.S.2节、第3.2.P.3节和第3.2.P.4节）。如第6章所述，应对分析方法各步骤（包括SST）进行详细描述，以便熟练的分析员能够执行分析测试。应按照 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> 指导原则提交验证数据。 <i>验证研究</i> 中使用的性能标准应包含在提交文件中。在某些情况下，根据预期目的（例如， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%BA%B6%E5%87%BA%E5%BA%A6%E6%A3%80%E6%B5%8B">溶出度检查</a> ）和/或所选技术， 也可以提交开发数据作为依据。</p><p>如第6章所述，如果制定了分析方法的ECs，则ECs应与支持性信息明确区分。在第3.2.S.4.3节和第3.2.P.5.3节中可提供其他开发和验证信息，以证明ECs及其报告类别的合理性。当提交文件中包含 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a> 中描述的其他 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 管理要素时， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%B3%E8%AF%B7%E4%BA%BA">申请人</a> 应遵循 <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a></i> 以及本文件第7章中描述的原则。</p><h2>10.2 增强方式的文件</h2><p>如果所用开发方式导致在分析方法控制策略中引入增强方式要素，则应证明合理性。</p><p>增强方式的性能特征和 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A5%E5%8F%97%E6%A0%87%E5%87%86">可接受标准</a> （例如，ATP中所述）以及其他要素（例如，MODR或PAR），应在申报资料的分析方法描述章节加以说明（例如，第3.2.S.4.2节和第3.2.P.5.2节）。如果提出了ECs，则ECs及支持性信息也应纳入分析方法描述中。采用增强方式时，不应导致在注册申报资料中提供更不详细的分析方法描述。</p><p>如果提出了ECs和相关报告类别，应提交基于风险的变更分类和相应的报告类别。对于作为ECs和不作为ECs的参数，应酌情提供依据（见第6章）。</p><p>应在注册申报资料的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法验证</a> 章节（例如，第3.2.S.4.3节和第3.2.P.5.3节）中总结并提交适当的分析方法风险评估和开发研究信息，以支持拟定的生命周期管理策略。</p><h2>10.3 多变量分析方法文件</h2><p>应根据模型的影响水平，提供与多变量分析方法相关的开发信息（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8/9/10_Q%26AsR4">ICH批准的ICH Q8/Q9/Q10 实施指导原则</a> ）。在申报资料的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%B7%A5%E8%89%BA%E5%BC%80%E5%8F%91">工艺开发</a> 章节（例如，第3.2.S.2.6节或第 3.2.P.2节）中提供生产开发研究或过程控制或检测中所用的多变量模型的开发信息。RTRT多变量模型的支持性开发信息可纳入到适当的分析方法验证或工艺开发章节。</p><p>在申报资料的验证信息章节（例如，第3.2.S.4.3节或第3.2.P.5.3节）中提供原料药或制剂放行所用的分析方法（包括RTRT）的验证信息。此外，这些章节还应包括用作参比分析方法的分析方法验证信息。模型开发、校正和验证信息可直接纳入到 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">CTD</a> 章节或附件中。</p><p>对于原料药或制剂 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 中所用的多变量模型（包括RTRT方法），对验证方案和结果的描述应包括：</p><ul><li>独立样本的验证集的描述；</li></ul><ul><li>多变量模型验证期间应满足的性能标准；</li></ul><ul><li>根据性能标准对 <i>模型验证</i> 结果的评价；</li></ul><ul><li>对模型性能标准和属性质量标准限度的关系的讨论；</li></ul><ul><li>对模型监测和维护的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E4%BD%93%E7%B3%BB">PQS</a> 要素的高度概述，例如用于确定模型样本数据适用性的诊断工具，以及发现离群值时采取的措施。</li></ul><p>用于RTRT的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的描述应包含在CTD第3.2.S.4.2节（ <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> ）或第3.2.P.5.2 节（制剂）中，通常包括：</p><ul><li>通过多变量分析方法测定的目标特性或属性以及期望的定量范围或限度；</li></ul><ul><li>测定原理和相关仪器操作参数的描述（例如，样品呈现形式、样品采集时间和测定频率）；</li></ul><ul><li>如何获得多变量模型校正数据的概述（例如，样品制备方法、参比分析方法）；</li></ul><ul><li>多变量模型的类型；</li></ul><ul><li>参比分析方法的描述或参比样品制备的概述；</li></ul><ul><li>将模型输出调整为报告值所需的任何计算。</li></ul><p>此外，第3.2.S.4.2节（原料药）或第3.2.P.5.2节（制剂）应包含任何作为已注册的RTRT替代 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> 的分析方法的描述。这些替代 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%96%B9%E6%B3%95%E9%AA%8C%E8%AF%81">分析方法的验证</a> 信息，应纳入到申报资料的验证信息章节（例如，第3.2.S.4.3节或第3.2.P.5.3节）。</p><h2>11. 术语表</h2><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> （ACCURACY）</b></p><p>分析方法的准确度指的是常规的真实值或认可的参比值与测定值或测定值集之间的相近程度。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> （ANALYTICAL PROCEDURE）</b></p><p>分析方法是指进行分析操作的方式。分析方法中应足够详细地描述每个分析检测所必需的步骤。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>分析方法属性（ANALYTICAL PROCEDURE ATTRIBUTE）</b></p><p>一种分析技术特有的性质，应在适当的限度、范围或分布内，以确保测定结果的预期质量。例如， <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%89%B2%E8%B0%B1">色谱法</a> 测定的分析方法属性可能包括峰对称因子和分离度。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> （ANALYTICAL PROCEDURE CONTROL STRATEGY）</b></p><p>基于对当前分析方法的理解而制定的一系列控制，用以确保分析方法的性能和测定结果的可靠。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>分析方法参数（ANALYTICAL PROCEDURE PARAMETER）</b></p><p>分析因素（包括试剂级别）或分析方法的操作条件，这些操作条件可连续变化（如，流速）或规定在可控的单一水平。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>分析方法验证策略（ANALYTICAL PROCEDURE VALIDATION STRATEGY）</b></p><p>分析方法验证策略描述用于验证的分析方法性能特征的选择。在策略中，开发研究期间收集的数据和系统 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%80%82%E7%94%A8%E6%80%A7%E8%AF%95%E9%AA%8C">适用性试验</a> （SST）可用于验证，并且可以预定义一组适当的验证试验。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>分析目标概况（ANALYTICAL TARGET PROFILE，ATP）</b></p><p>用以描述分析测定预期目的性能特征和预期性能标准的前瞻性总结。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>校正模型（CALIBRATION MODEL）</b></p><p>一种基于已知样本分析检测的模型，该模型将输入数据与关注属性值（即模型输出）相关联。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A7%E5%88%B6%E7%AD%96%E7%95%A5">控制策略</a> （CONTROL STRATEGY）</b></p><p>根据当前对产品和工艺的理解而产生的一系列保证工艺性能和产品质量的有计划的控制。这些控制可包括与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8E%9F%E6%96%99%E8%8D%AF">原料药</a> 和制剂的物料与组分、厂房和设备运行条件、过程控制、成品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 相关的参数和特性，以及相应的监控方法和频次。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%85%B3%E9%94%AE%E8%B4%A8%E9%87%8F%E5%B1%9E%E6%80%A7">关键质量属性</a> （CRITICAL QUALITY ATTRIBUTE， CQA）</b></p><p>指产品的物理、化学、生物或微生物性质或特征，应在适当的限度、 范围或分布之内，以确保预期的产品质量。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:CTD">CTD</a></b></p><p>通用技术文档（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=M4Q(R1)_Quality">ICH M4Q</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:DOE">DoE</a></b></p><p>实验设计。</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%97%A2%E5%AE%9A%E6%9D%A1%E4%BB%B6">既定条件</a> （ESTABLISHED CONDITIONS，EC）</b></p><p>EC是确保产品质量所必需的具有法律约束力的信息。因此，EC的任何变更都必须提交给监管机构。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a></i> ）</p><p><b>中间精密度（INTERMEDIATE PRECISION）</b></p><p>中间精密度是指实验室内变异性。考虑的因素应包括变异度潜在的来源，例如不同天、不同环境条件、不同分析人员、不同仪器。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>知识管理（KNOWLEDGE MANAGEMENT）</b></p><p>收集、分析、储存和传递与产品、生产工艺和组分相关信息的系统性方法。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q10_Pharmaceutical_Quality_System">ICH Q10</a></i> ）</p><p><b>方法可操作设计区域（METHOD OPERABLE DESIGN REGION， MODR）</b></p><p>分析方法参数范围的组合，在该范围内操作，能达到 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 的性能标准，并且能够确保测定结果的质量。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>持续监测（ONGOING MONITORING）</b></p><p>分析方法性能数据的收集和评价，以确保整个分析方法 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内测定结果的可靠。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>PACMP</b></p><p>批准后 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%8F%98%E6%9B%B4%E7%AE%A1%E7%90%86">变更管理</a> 方案。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q12_Technical_and_Regulatory_Considerations_for_Pharmaceutical_Product_Lifecycle_Management_201911">ICH Q12</a></i> ）</p><p><b>性能特征（PERFORMANCE CHARACTERISTIC）</b></p><p>确保测定结果可靠的一种特性，该特性与分析技术无关。通常可能包括准确度、精密度、专属性/选择性和范围。既往 <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> 版本称之为验证指标。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>性能标准（PERFORMANCE CRITERION）</b></p><p>描述一个给定性能特征的数值范围、限度或期望状态的可接受标准，以确保测定结果的可靠。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>平台分析方法（PLATFORM ANALYTICAL PROCEDURE）</b></p><p>适用于测定不同产品的质量属性而不需要对其操作条件、系统适用性和报告结构进行重大改变的分析方法。这类分析方法可用于分析属性非常相似的分子，相关属性即为平台分析方法所测定。 <b>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）</b></p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%B2%BE%E5%AF%86%E5%BA%A6">精密度</a> （PRECISION）</b></p><p>分析方法的精密度是指在规定条件下，对均质样品多次取样检测得到一系列检测结果之间的接近程度（离散程度）。精密度可以从三个层次考虑：重复性，中间精密度和重现性。</p><p>分析方法的精密度通常以一系列多次检测结果的方差、标准 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%81%8F%E5%B7%AE">偏差</a> 或变异系数来表示。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>经证实的分析方法可接受范围（PROVEN ACCEPTABLE RANGE FOR ANALYTICAL PROCEDURES， PAR）</b></p><p>是指分析方法参数范围。在此范围内操作，同时保持其他参数不变，可使分析测定结果符合相关性能标准。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%A3%8E%E9%99%A9%E7%AE%A1%E7%90%86">质量风险管理</a> （QUALITY RISK MANAGEMENT， QRM）</b></p><p>在整个产品 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内评估、控制、沟通和回顾药品质量风险的系统化过程。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q9_R1_Quality_Risk_Management_20230118">ICH Q9</a></i> ）</p><p><b>范围（RANGE）</b></p><p>分析方法的范围是可报告的最低结果和最高结果之间的区间，在此区间内分析方法具有适当水平的精密度、 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%87%86%E7%A1%AE%E5%BA%A6">准确度</a> 及响应。 <b>（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）</b></p><p><b>可报告范围（REPORTABLE RANGE）</b></p><p>分析方法的可报告范围包括从最低至最高可报告结果的所有值，这些值具有适当水平的精密度和准确度。可报告范围的单位通常与 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E8%B4%A8%E9%87%8F%E6%A0%87%E5%87%86">质量标准</a> 中的可接受标准相同。（ <b><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></b> ）</p><p><b>工作范围（WORKING RANGE）</b></p><p><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%88%86%E6%9E%90%E6%96%B9%E6%B3%95">分析方法</a> 工作范围是分析方法提供有意义结果的最低浓度至最高范围。样品制备前对应工作范围（样品工作范围）与分析仪器检测前对应工作范围（仪器工作范围）可能不同。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>实时 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%94%BE%E8%A1%8C">放行</a> 检测（REAL TIME RELEASE TESTING， RTRT）</b></p><p>指根据工艺数据评价并确保中间产品和/或成品质量的能力，通常包括已测得物料属性和工艺控制的有效结合。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q8_Pharmaceutical_Development">ICH Q8</a></i> ）</p><p><b>参比物质（REFERENCE MATERIAL）</b></p><p>一种经过适当标定的物质，在一个或多个定义的属性方面具有足够均匀性和稳定性，且确定适用于预期目的。参比物质可能包括国家/国际标准物质、药典标准物质、内部一级/二级参比物质。（ <a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a> ）</p><p><b>重复性（REPEATABILITY）</b></p><p>重复性是指在同样操作条件下，短时间间隔内测量的精密度。重复性也称为测量精密度（Intra-assay precision）。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>可报告结果（REPORTABLE RESULT）</b></p><p>经计算或处理，并考虑所述重复性测量后，由分析方法得到结果（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>重现性（REPRODUCIBILITY）</b></p><p>重现性是指不同试验室之间测量的精密度（ <b>如</b> ：试验室间研究，通常用于方法的标准化）。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>响应（RESPONSE）</b></p><p>分析方法的响应是分析方法（在给定范围内）获得信号的能力，通过某些已知的数学函数能将这些获得的信号与样品中被分析物浓度（含量）或活性有效联系起来。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E5%86%8D%E9%AA%8C%E8%AF%81">再验证</a> （REVALIDATION）</b></p><p>证明产品、工艺或分析方法本身变更后，分析方法仍符合其预期目的。再验证可涉及所有（完整再验证）或部分（部分再验证）的性能特征。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>耐用性（ROBUSTNESS）</b></p><p>分析方法的耐用性是衡量其在正常使用时符合预期性能标准的能力。通过有意变化分析方法参数来检验耐用性。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>样品适用性评估（SAMPLE SUITABILITY ASSESSMENT）</b></p><p>如果样品的检测响应符合分析方法属性预设的 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E6%8E%A5%E5%8F%97%E6%A0%87%E5%87%86">可接受标准</a> ，则认为样品或样品制备是适用的，其中分析方法属性预设的可接受标准已开发用于验证的分析方法。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>专属性/选择性（SPECIFICITY/SELECTIVITY）</b></p><p>在用指定分析方法测定某一分析物时，专属性和选择性均用于描述其他物质的干扰程度。专属性通常用于描述最终状态，即分析方法可以明确检测目标分析物。选择性则是一个相对术语， 用于描述混合物或基质中特定被分析物可被检测且不受具有类似行为的其他组分干扰的程度。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%80%82%E7%94%A8%E6%80%A7%E8%AF%95%E9%AA%8C">系统适用性试验</a> （SYSTEM SUITABILITY TEST，SST）</b></p><p>开发系统适用性试验用于确认检测系统及与分析方法相关的分析操作适用于预期目的，并提高对于分析方法性能不可接受情况的可检测性。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b><a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E9%AA%8C%E8%AF%81">验证</a> 研究（VALIDATION STUDY）</b></p><p>对先验知识、数据或专项实验（即验证试验）的评价，以确定分析方法适用其预期目的。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>验证试验（VALIDATION TEST）</b></p><p>验证试验是为验证分析方法适用其预期目的而设计的专项实验。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b><u>多变量术语表</u> （MULTIVARIATE GLOSSARY）</b></p><p><b>校正集（CALIBRATION SET）</b></p><p>一组已知特征和分析检测结果相匹配的数据。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>数据转换（DATA TRANSFORMATION）</b></p><p>为获得与输出数据更好的相关性并简化模型结构，而对模型输入数据进行的数学处理。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>独立样本（INDEPENDENT SAMPLE）</b></p><p>独立样本是指未包含在多变量模型校正集中的样品。独立样本与校正样本可来源于同一批次。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>内部测试（INTERNAL TESTING）</b></p><p>内部测试是检查特定样品数据经过模型处理后预测是否正确（定性或定量）的过程。</p><p>内部测试用于确定潜变量的最佳数量、估算标准误差及检测潜在离群值。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>内部测试集（INTERNAL TEST SET）</b></p><p>从样品中获得的一组数据，这些样品涵盖的物理化学特性变异范围与用于构建校正集样品类似 。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>潜变量（LATENT VARIABLES）</b></p><p>与测量变量直接相关经由数学推导而得的变量，潜变量用于进一步分析处理。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>模型维护（MODEL MAINTENANCE）</b></p><p>在多变量模型全 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">生命周期</a> 内用于确保持续的模型性能的过程，通常包括 <a href="https://lib.shilinx.com/wiki/index.php?title=%E5%88%86%E7%B1%BB:%E7%A6%BB%E7%BE%A4%E5%80%BC">离群值</a> 诊断和由模型再开发或维护计划变更引起的动作。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>模型验证（MODEL VALIDATION）</b></p><p>通过使用独立的试验数据挑战模型，并将结果与预定性能标准进行比较，从而确定模型适用性的过程。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>多变量分析方法（MULTIVARIATE ANALYTICAL PROCEDURE）</b></p><p>通过使用多于一个输入变量的多变量校正模型而确定结果的分析方法。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>离群值诊断（OUTLIER DIAGNOSTIC）</b></p><p>能够在多变量分析方法中识别异常或非典型数据的测试。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>参比分析方法（REFERENCE ANALYTICAL PROCEDURE）</b></p><p>单独的分析方法，用于获得多变量分析方法中校正样本和验证样品的参比值。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><p><b>参比样品（REFERENCE SAMPLE）</b></p><p>是指代表测试样品，具有关注属性的已知值，并用于校准的样品（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q14_Analytical_Procedure_Development_20231101">ICH Q14</a></i> ）</p><p><b>验证集（VALIDATION SET）</b></p><p>用于对校正模型的性能进行独立评估的一组数据。（ <i><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2</a></i> ）</p><h2>12. 参考资料</h2><p><a href="https://lib.shilinx.com/wiki/index.php?title=ICH_Q2_R2_Validation_of_Analytical_Procedures_20231101">ICH Q2 分析方法验证</a></p><p><a
